# React Output Question 07: useRef can hold the latest value for async callbacks

## Basic Info
- Category: Hooks / useRef
- Difficulty: Intermediate
- Tags: useRef, closure, latest value
- Type: Read React code and write the output

## Description
和上一题类似，但这次使用 `ref` 保存最新值。点击一次后，1 秒后会输出什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const latest = React.useRef(count);
  latest.current = count;

  const handleClick = () => {
    setTimeout(() => {
      console.log(latest.current);
    }, 1000);
    setCount(c => c + 1);
  };

  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## Answer
It logs 1.

## Key Points
- refs are mutable containers
- reading ref.current avoids stale state in many async cases

## Detailed Explanation
虽然定时器回调本身依然属于旧闭包，但它访问的是 `latest.current`，而 `ref` 对象本身是稳定不变的。点击后组件重新渲染，并把 `latest.current` 更新成 `1`，所以等 1 秒后回调真正执行时，读取到的是最新值 `1`。
