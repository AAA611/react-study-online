# React 输出题 76：useCallback 可以稳定函数 props，减少记忆化子组件重渲染

## 基本信息
- 分类：性能优化 / useCallback
- 难度：中级
- 标签：useCallback, React.memo, 函数稳定
- 题型：阅读 React 代码并写出输出

## 题目描述
下面这段代码里，点击按钮后子组件会不会再次渲染？

## 代码
```jsx
const Child = React.memo(function Child({ onAdd }) {
  console.log("child render");
  return <button onClick={onAdd}>child</button>;
});

function App() {
  const [count, setCount] = React.useState(0);
  const handleAdd = React.useCallback(() => {
    setCount((value) => value + 1);
  }, []);

  console.log("app render", count);

  return <Child onAdd={handleAdd} />;
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render`。点击按钮后只会输出 `app render 1`，不会再次输出 `child render`。

## 考察点
- useCallback 可以缓存函数引用
- 稳定函数 props 配合 React.memo 可以跳过重复渲染
- 函数式更新常用于减少 callback 对 state 的依赖

## 详细讲解
第一次渲染时，`App` 和 `Child` 都会执行，因此输出 `app render 0` 和 `child render`。这里的关键在于 `handleAdd` 通过 `useCallback(..., [])` 被缓存了下来，并且内部使用的是函数式更新，所以它不依赖外部 `count`。点击按钮后，`App` 会重新渲染并输出 `app render 1`，但传给 `Child` 的 `onAdd` 还是同一个函数引用。React.memo 浅比较通过，于是子组件被跳过，不会再次输出 `child render`。
