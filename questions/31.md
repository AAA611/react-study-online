# React 输出题 31：空依赖 effect 会保留旧函数

## 基本信息
- 分类：Hooks / 依赖数组
- 难度：中级
- 标签：依赖数组, 旧闭包, 函数
- 题型：阅读 React 代码并写出输出

## 题目描述
1 秒后控制台会打印什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const log = () => {
    console.log(count);
  };

  React.useEffect(() => {
    setTimeout(log, 1000);
  }, []);

  React.useEffect(() => {
    setCount(5);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
最终打印的是 `0`。

## 考察点
- 这个定时器保存的是第一次渲染时的 `log`
- 空依赖数组会把这次 effect 的初始化逻辑固定下来

## 详细讲解
第一个 effect 只会在首次挂载后执行一次，因此定时器里保存的是第一次渲染时创建的 `log` 函数。那个函数闭包中的 `count` 是 `0`，所以即使后面 state 被改成 `5`，1 秒后打印的仍然是 `0`。
