# React Output Question 31: Empty dependency effect can keep an old function

## Basic Info
- Category: Hooks / dependency array
- Difficulty: Intermediate
- Tags: dependency array, stale closure, function
- Type: Read React code and write the output

## Description
1 秒后控制台会打印什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const log = () => {
    console.log(count);
  };

  React.useEffect(() => {
    setTimeout(log, 1000);
  }, []);

  React.useEffect(() => {
    setCount(5);
  }, []);

  return null;
}

render(<App />);
```

## Answer
It logs 0.

## Key Points
- the timeout stores the first render version of log
- empty dependency arrays freeze that effect setup

## Detailed Explanation
第一个 effect 只会在首次挂载后执行一次，因此定时器里保存的是第一次渲染时创建的 `log` 函数。那个函数闭包中的 `count` 是 `0`，所以即使后面 state 被改成 `5`，1 秒后打印的仍然是 `0`。
