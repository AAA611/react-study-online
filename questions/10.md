# React Output Question 10: useCallback caches function identity, not execution

## Basic Info
- Category: Performance / useCallback
- Difficulty: Intermediate
- Tags: useCallback, effect, function identity
- Type: Read React code and write the output

## Description
点击前和点击后，控制台分别会输出什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const log = React.useCallback(() => {
    console.log("callback", count);
  }, [count]);

  React.useEffect(() => {
    log();
  }, [log]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## Answer
Output: render 0 -> callback 0, then after one click render 1 -> callback 1.

## Key Points
- useCallback creates a new function when dependencies change
- effect depends on function identity here

## Detailed Explanation
`useCallback` 缓存的是函数引用，而不是函数执行结果。当 `count` 从 `0` 变成 `1` 时，依赖发生变化，React 会返回一个新的 `log` 函数。由于 `effect` 依赖的是 `log`，这个引用变化会让 effect 再执行一次，因此点击后会继续输出 `callback 1`。
