# React 输出题 10：useCallback 缓存的是函数引用，不是执行结果

## 基本信息
- 分类：性能 / useCallback
- 难度：中级
- 标签：useCallback, effect, 函数引用
- 题型：阅读 React 代码并写出输出

## 题目描述
点击前和点击后，控制台分别会输出什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const log = React.useCallback(() => {
    console.log("callback", count);
  }, [count]);

  React.useEffect(() => {
    log();
  }, [log]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
输出顺序是：首次渲染打印 `render 0 -> callback 0`；点击一次后打印 `render 1 -> callback 1`。

## 考察点
- 依赖变化时，useCallback 会生成新的函数引用
- 这里的 effect 依赖的是函数引用是否变化

## 详细讲解
`useCallback` 缓存的是函数引用，而不是函数执行结果。当 `count` 从 `0` 变成 `1` 时，依赖发生变化，React 会返回一个新的 `log` 函数。由于 `effect` 依赖的是 `log`，这个引用变化会让 effect 再执行一次，因此点击后会继续输出 `callback 1`。
