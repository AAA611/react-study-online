# React 输出题 77：useCallback 依赖变化时，记忆化子组件仍然会重新渲染

## 基本信息
- 分类：性能优化 / useCallback
- 难度：中级
- 标签：useCallback, 依赖项, 函数引用
- 题型：阅读 React 代码并写出输出

## 题目描述
`handleClick` 明明用了 `useCallback`，为什么点击后子组件还是重新渲染？

## 代码
```jsx
const Child = React.memo(function Child({ onClick }) {
  console.log("child render");
  return <button onClick={onClick}>child</button>;
});

function App() {
  const [count, setCount] = React.useState(0);
  const handleClick = React.useCallback(() => {
    setCount(count + 1);
  }, [count]);

  console.log("app render", count);

  return <Child onClick={handleClick} />;
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render`。点击一次后会继续输出 `app render 1`、`child render`。

## 考察点
- useCallback 不是“永远不变”，而是依赖变了就会生成新函数
- callback 依赖 state 时，state 变化会导致函数引用变化
- 优化时需要区分“用了 useCallback”和“函数真的稳定”这两件事

## 详细讲解
`handleClick` 虽然写成了 `useCallback`，但它依赖了 `count`，因为内部直接使用了 `count + 1`。这样一来，每次 `count` 变化时，`handleClick` 都会生成新的函数引用。点击按钮后，`App` 重新渲染并输出 `app render 1`，新的 `onClick` 传给 `Child` 后，React.memo 浅比较会发现函数变了，所以子组件仍然会重新执行，再次输出 `child render`。
