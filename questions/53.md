# React 输出题 53：`key` 一样也不能跨组件类型复用

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, key, 组件类型
- 题型：阅读 React 代码并写出输出

## 题目描述
下面这段代码里，两个节点的 `key` 都是 `"same"`。React 会复用它们吗？

## 代码
```jsx
function Foo() {
  const [value] = React.useState(10);
  console.log("Foo", value);
  return null;
}

function Bar() {
  const [value] = React.useState(20);
  console.log("Bar", value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag ? <Bar key="same" /> : <Foo key="same" />;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `Foo 10`，再打印 `Bar 20`。

## 考察点
- `key` 只是在同层级比较时帮助 React 找到对应节点
- `key` 相同不代表不同类型组件一定会复用
- React 仍然会先看类型，再决定是否复用实例

## 详细讲解
第一次渲染时输出 `Foo 10`。更新后，虽然新旧节点的 `key` 都是 `"same"`，但它们的类型一个是 `Foo`，一个是 `Bar`。React diff 的前提是“先匹配位置和 key，再比较类型是否可复用”。当类型不同的时候，`key` 并不能强行让 React 复用旧实例，所以这里仍然会卸载 `Foo`，重新挂载 `Bar`，最终输出 `Bar 20`。
