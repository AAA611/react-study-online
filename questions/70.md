# React 输出题 70：更新时 useLayoutEffect 和 useEffect 的 cleanup 时机不同

## 基本信息
- 分类：Hooks / cleanup
- 难度：中级
- 标签：useLayoutEffect, useEffect, cleanup
- 题型：阅读 React 代码并写出输出

## 题目描述
当 `count` 从 `0` 变成 `1` 时，两个 Hook 的 cleanup 和新回调会按什么顺序执行？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useLayoutEffect(() => {
    console.log("layout effect", count);

    return () => {
      console.log("layout cleanup", count);
    };
  }, [count]);

  React.useEffect(() => {
    console.log("effect", count);

    if (count === 0) {
      setCount(1);
    }

    return () => {
      console.log("cleanup", count);
    };
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> layout effect 0 -> effect 0 -> render 1 -> layout cleanup 0 -> layout effect 1 -> cleanup 0 -> effect 1`。

## 考察点
- 更新阶段时，useLayoutEffect 的 cleanup 和新回调会更早执行
- useEffect 的 cleanup 与新回调发生在更靠后的时机
- cleanup 读到的仍然是旧闭包中的值

## 详细讲解
第一次渲染后，先执行 `useLayoutEffect`，输出 `layout effect 0`，再执行 `useEffect`，输出 `effect 0`。因为 `count === 0`，`useEffect` 会把状态更新成 `1`。第二轮渲染先输出 `render 1`。提交更新时，React 会先处理旧的 `useLayoutEffect` 清理逻辑，所以输出 `layout cleanup 0`，随后立即执行新的 `useLayoutEffect`，输出 `layout effect 1`。之后再进入普通 effect 的清理和执行阶段，于是输出 `cleanup 0`，最后输出新的 `effect 1`。这正体现了 `useLayoutEffect` 比 `useEffect` 更早参与更新提交流程。
