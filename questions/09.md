# React Output Question 09: Parent rerender does not always recompute child memo value

## Basic Info
- Category: Performance / useMemo
- Difficulty: Junior
- Tags: parent child, useMemo, rendering
- Type: Read React code and write the output

## Description
父组件点击后重新渲染，子组件里的 `memo` 计算会不会重新执行？

## Code
```jsx
function Child({ label }) {
  const value = React.useMemo(() => {
    console.log("memo", label);
    return label + "!";
  }, [label]);

  console.log("child render", value);
  return null;
}

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Child label="React" />
    </div>
  );
}

render(<App />);
```

## Answer
First render: app render 0, memo React, child render React!. After one click: app render 1, child render React!.

## Key Points
- child still rerenders when parent rerenders
- memo callback is skipped if dependency is stable

## Detailed Explanation
父组件状态变化后，子组件函数本身还是会重新执行一次，所以 `child render` 会再次出现。但 `label` 依然是同一个字符串 `React`，因此 `useMemo` 会直接返回缓存结果，不会再次执行计算函数，也就不会再次输出 `memo React`。
