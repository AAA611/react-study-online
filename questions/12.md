# React 输出题 12：React.memo 遇到每次都是新对象的 props

## 基本信息
- 分类：性能 / React.memo
- 难度：中级
- 标签：React.memo, 对象引用, 浅比较
- 题型：阅读 React 代码并写出输出

## 题目描述
为什么 props 内容看起来没变，子组件还是会重新渲染？

## 代码
```jsx
const MemoChild = React.memo(function Child({ config }) {
  console.log("child render", config.value);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <MemoChild config={{ value: 1 }} />
    </div>
  );
}

render(<App />);
```

## 参考答案
首次渲染会打印 `app render 0`、`child render 1`。点击一次后会再次打印 `app render 1`、`child render 1`。

## 考察点
- 对象字面量每次都会创建新的引用
- 遇到新的对象引用时，React.memo 的浅比较会判定 props 已变化

## 详细讲解
虽然 `config.value` 一直都是 `1`，但 JSX 里的对象字面量在每次渲染时都会创建一个新的对象引用。`React.memo` 只做浅比较，看到新旧 `config` 不是同一个引用，就会认为 props 发生了变化，因此子组件仍然会重新渲染。
