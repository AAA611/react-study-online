# React Output Question 12: React.memo with a new object prop each time

## Basic Info
- Category: Performance / React.memo
- Difficulty: Intermediate
- Tags: React.memo, object reference, shallow compare
- Type: Read React code and write the output

## Description
为什么 props 内容看起来没变，子组件还是会重新渲染？

## Code
```jsx
const MemoChild = React.memo(function Child({ config }) {
  console.log("child render", config.value);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <MemoChild config={{ value: 1 }} />
    </div>
  );
}

render(<App />);
```

## Answer
First render: app render 0, child render 1. After one click: app render 1, child render 1.

## Key Points
- object literals create new references
- React.memo shallow compare fails on a new object reference

## Detailed Explanation
虽然 `config.value` 一直都是 `1`，但 JSX 里的对象字面量在每次渲染时都会创建一个新的对象引用。`React.memo` 只做浅比较，看到新旧 `config` 不是同一个引用，就会认为 props 发生了变化，因此子组件仍然会重新渲染。
