# React Output Question 03: Functional updates run one after another

## Basic Info
- Category: State / functional updates
- Difficulty: Junior
- Tags: functional update, batching, state
- Type: Read React code and write the output

## Description
这段代码里 `effect` 执行完成后，最终渲染输出是什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    setCount(c => c + 1);
    setCount(c => c + 1);
  }, []);

  console.log("render", count);
  return null;
}

render(<App />);
```

## Answer
Output: render 0, then render 2.

## Key Points
- functional updates receive latest queued state
- batched updates can still accumulate

## Detailed Explanation
第一次渲染时输出 `render 0`。`effect` 中的第一条函数式更新会把 `0` 变成 `1`，第二条函数式更新拿到的已经是上一次更新后的值 `1`，再把它变成 `2`。虽然 React 会把这两次更新批处理到同一轮提交中，但最终状态仍然会累加到 `2`，所以第二次渲染输出 `render 2`。
