# React 输出题 03：函数式更新会按顺序依次执行

## 基本信息
- 分类：状态 / 函数式更新
- 难度：初级
- 标签：函数式更新, 批处理, 状态
- 题型：阅读 React 代码并写出输出

## 题目描述
这段代码里 `effect` 执行完成后，最终渲染输出是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    setCount(c => c + 1);
    setCount(c => c + 1);
  }, []);

  console.log("render", count);
  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `render 0`，再打印 `render 2`。

## 考察点
- 函数式更新能拿到队列中的最新状态
- 即使被批处理，多个更新依然可以正确累加

## 详细讲解
第一次渲染时输出 `render 0`。`effect` 中的第一条函数式更新会把 `0` 变成 `1`，第二条函数式更新拿到的已经是上一次更新后的值 `1`，再把它变成 `2`。虽然 React 会把这两次更新批处理到同一轮提交中，但最终状态仍然会累加到 `2`，所以第二次渲染输出 `render 2`。
