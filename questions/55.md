# React 输出题 55：节点移动到不同父元素下时即使 `key` 一样也会重建

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, key, 父节点
- 题型：阅读 React 代码并写出输出

## 题目描述
同一个 `Child` 从左边的 `div` 挪到右边的 `section`，而且 `key` 还是同一个。控制台会输出什么？

## 代码
```jsx
function Child() {
  const [count] = React.useState(7);
  console.log("child", count);
  return null;
}

function App() {
  const [move, setMove] = React.useState(false);

  React.useEffect(() => {
    setMove(true);
  }, []);

  return (
    <>
      <div>{move ? null : <Child key="only" />}</div>
      <section>{move ? <Child key="only" /> : null}</section>
    </>
  );
}

render(<App />);
```

## 参考答案
会打印两次 `child 7`。

## 考察点
- `key` 的作用范围只在同一个父节点的同层子元素中
- 节点移动到不同父节点下时，React 不会跨父节点复用实例
- 看起来像“移动”，本质上通常是旧节点卸载、新节点挂载

## 详细讲解
第一次渲染时，`Child` 在左侧的 `div` 里，所以输出一次 `child 7`。更新后，它从 `div` 里消失，出现在 `section` 里。虽然两个地方的 `key` 都是 `"only"`，但它们并不在同一个父节点下，React 不会跨父节点去做实例复用。因此这里不是把旧组件“搬过去”，而是把左边的旧实例卸载，再在右边创建一个新实例，所以又会重新初始化 state，再打印一次 `child 7`。
