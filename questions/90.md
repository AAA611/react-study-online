# React 输出题 90：给已有子组件临时套一层包装会打断原有复用路径

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：高级
- 标签：树结构, 包装层, 复用路径
- 题型：阅读 React 代码并写出输出

## 题目描述
第一次直接返回两个兄弟节点，第二次把它们包进一个 `section`。控制台输出是什么？

## 代码
```jsx
function Card({ label }) {
  const [value] = React.useState(label);
  console.log("card", label, value);
  return null;
}

function App() {
  const [wrapped, setWrapped] = React.useState(false);

  React.useEffect(() => {
    setWrapped(true);
  }, []);

  return wrapped ? (
    <section>
      <Card label="A" />
      <Card label="B" />
    </section>
  ) : (
    <>
      <Card label="A" />
      <Card label="B" />
    </>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `card A A`、`card B B`，更新后再打印 `card A A`、`card B B`。

## 考察点
- React diff 比较的是整棵返回树的结构
- 外层结构一旦变化，原来的复用路径可能整体失效
- “只是加了一层包装”在 diff 看来未必是小改动

## 详细讲解
第一次渲染时，`App` 直接返回两个 `Card` 兄弟节点，所以依次输出 `card A A` 和 `card B B`。更新后，顶层结构从 `Fragment` 下的两个子节点，变成了一个 `section`，而两个 `Card` 被放到了 `section` 里面。对于 React 来说，这不是简单地“原来那两个 `Card` 还在原地”，而是整条返回树的上层结构发生了变化。原来的复用路径被打断，新的 `Card` 会重新挂载，因此会再次输出 `card A A` 和 `card B B`。
