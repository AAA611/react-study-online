# React Output Question 50: Two state updates in one effect usually produce one final rerender

## Basic Info
- Category: State / batching
- Difficulty: Intermediate
- Tags: batching, multiple state values, effect
- Type: Read React code and write the output

## Description
effect 中同时更新 `a` 和 `b`，最终日志会是什么？

## Code
```jsx
function App() {
  const [a, setA] = React.useState(0);
  const [b, setB] = React.useState(0);

  console.log("render", a, b);

  React.useEffect(() => {
    setA(1);
    setB(2);
  }, []);

  return null;
}

render(<App />);
```

## Answer
Output: render 0 0, then render 1 2.

## Key Points
- multiple state updates in the same effect are typically batched together

## Detailed Explanation
首次渲染时 `a` 和 `b` 都是 `0`，所以先输出 `render 0 0`。挂载后的 effect 同时触发两个状态更新，React 通常会把它们批处理到同一轮提交里，因此只额外渲染一次，最终输出 `render 1 2`。
