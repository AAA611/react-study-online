# React 输出题 50：一个 effect 里的两次 state 更新通常只触发一次最终重渲染

## 基本信息
- 分类：状态 / 批处理
- 难度：中级
- 标签：批处理, 多个状态值, effect
- 题型：阅读 React 代码并写出输出

## 题目描述
effect 中同时更新 `a` 和 `b`，最终日志会是什么？

## 代码
```jsx
function App() {
  const [a, setA] = React.useState(0);
  const [b, setB] = React.useState(0);

  console.log("render", a, b);

  React.useEffect(() => {
    setA(1);
    setB(2);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 0 -> render 1 2`。

## 考察点
- 同一个 effect 里的多个 state 更新通常会被合并批处理

## 详细讲解
首次渲染时 `a` 和 `b` 都是 `0`，所以先输出 `render 0 0`。挂载后的 effect 同时触发两个状态更新，React 通常会把它们批处理到同一轮提交里，因此只额外渲染一次，最终输出 `render 1 2`。
