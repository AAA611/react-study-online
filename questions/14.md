# React Output Question 14: Stable keys preserve the right component identity

## Basic Info
- Category: Lists / key
- Difficulty: Intermediate
- Tags: stable key, identity, list
- Type: Read React code and write the output

## Description
如果 `key` 改成稳定值，输出会发生什么变化？

## Code
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["A", "B"]);

  React.useEffect(() => {
    setList(["B", "A"]);
  }, []);

  return list.map((item) => <Item key={item} label={item} />);
}

render(<App />);
```

## Answer
Logs: A A, B B, then B B, A A.

## Key Points
- stable keys keep the correct identity
- moving an item is different from replacing an item

## Detailed Explanation
这里的 `key` 直接使用了元素本身，因此 React 能正确知道哪一个节点是 `A`，哪一个节点是 `B`。列表顺序变化时，组件实例会跟着 key 一起移动，所以 state 依然和正确的元素对应，不会发生错位。
