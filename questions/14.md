# React 输出题 14：稳定 key 能保留正确的组件身份

## 基本信息
- 分类：列表 / key
- 难度：中级
- 标签：稳定 key, 身份标识, 列表
- 题型：阅读 React 代码并写出输出

## 题目描述
如果 `key` 改成稳定值，输出会发生什么变化？

## 代码
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["A", "B"]);

  React.useEffect(() => {
    setList(["B", "A"]);
  }, []);

  return list.map((item) => <Item key={item} label={item} />);
}

render(<App />);
```

## 参考答案
日志顺序是：先打印 `A A`、`B B`，重排后再打印 `B B`、`A A`。

## 考察点
- 稳定的 key 能保留正确的组件身份
- 移动一个元素和替换一个元素在 React 看来不是同一件事

## 详细讲解
这里的 `key` 直接使用了元素本身，因此 React 能正确知道哪一个节点是 `A`，哪一个节点是 `B`。列表顺序变化时，组件实例会跟着 key 一起移动，所以 state 依然和正确的元素对应，不会发生错位。
