# React 输出题 47：状态拆分不代表组件只执行一部分

## 基本信息
- 分类：渲染模型 / 状态
- 难度：初级
- 标签：状态拆分, 组件执行, 渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮修改 `a` 时，和 `b` 相关的日志会不会再次出现？

## 代码
```jsx
function App() {
  const [a, setA] = React.useState(0);
  const [b] = React.useState(100);

  console.log("render", a, b);

  return <button onClick={() => setA(n => n + 1)}>{a}</button>;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 100`，点击一次后打印 `render 1 100`。

## 考察点
- 组件一旦重渲染，整个函数体都会重新执行

## 详细讲解
即使只有 `a` 改变了，函数组件重新渲染时依然会重新执行整个函数体，而不是只执行和 `a` 相关的局部代码。所以日志里仍然会再次看到 `b` 的值。
