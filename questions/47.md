# React Output Question 47: Splitting state does not mean partial component execution

## Basic Info
- Category: Render model / state
- Difficulty: Junior
- Tags: state split, component execution, render
- Type: Read React code and write the output

## Description
点击按钮修改 `a` 时，和 `b` 相关的日志会不会再次出现？

## Code
```jsx
function App() {
  const [a, setA] = React.useState(0);
  const [b] = React.useState(100);

  console.log("render", a, b);

  return <button onClick={() => setA(n => n + 1)}>{a}</button>;
}

render(<App />);
```

## Answer
Output: render 0 100, then after one click render 1 100.

## Key Points
- the whole component function runs again on rerender

## Detailed Explanation
即使只有 `a` 改变了，函数组件重新渲染时依然会重新执行整个函数体，而不是只执行和 `a` 相关的局部代码。所以日志里仍然会再次看到 `b` 的值。
