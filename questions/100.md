# React 输出题 100：useLayoutEffect 的更新会在 useEffect 看到新值之前先触发下一轮渲染

## 基本信息
- 分类：执行顺序 / layout effect
- 难度：高级
- 标签：useLayoutEffect, useEffect, 更新阶段
- 题型：阅读 React 代码并写出输出

## 题目描述
首次挂载后，这段代码的完整输出顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useLayoutEffect(() => {
    console.log("layout", count);

    if (count === 0) {
      setCount(1);
    }
  }, [count]);

  React.useEffect(() => {
    console.log("effect", count);
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> layout 0 -> effect 0 -> render 1 -> layout 1 -> effect 1`。

## 考察点
- 每一轮里 `useLayoutEffect` 都早于 `useEffect`
- `useLayoutEffect` 中的更新会触发下一轮渲染
- `useEffect` 读取的是各自那一轮提交对应的 state 值

## 详细讲解
首次渲染先输出 `render 0`。提交后，`useLayoutEffect` 先执行，因此输出 `layout 0`，并在 `count===0` 时触发 `setCount(1)`。这一轮的 `useEffect` 仍然会基于当前这轮提交执行，所以先输出 `effect 0`。之后 React 进入下一轮渲染，输出 `render 1`。新的提交完成后，再次先执行 `useLayoutEffect` 输出 `layout 1`，最后执行 `useEffect` 输出 `effect 1`。这题的关键是理解：layout/effect 都是“每一轮提交”的一部分，但 layout 的时机更早。
