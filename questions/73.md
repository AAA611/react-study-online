# React 输出题 73：React.memo 遇到新的对象 props 仍然会重新渲染

## 基本信息
- 分类：性能优化 / React.memo
- 难度：中级
- 标签：React.memo, 对象引用, 浅比较
- 题型：阅读 React 代码并写出输出

## 题目描述
子组件已经用了 `React.memo`，为什么点击后还是会重新渲染？

## 代码
```jsx
const Child = React.memo(function Child({ config }) {
  console.log("child render", config.name);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <Child config={{ name: "React" }} />
    </>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render React`。点击一次后会继续输出 `app render 1`、`child render React`。

## 考察点
- React.memo 只做浅比较，不会深比较对象内容
- JSX 中的对象字面量每次渲染都会创建新引用
- 想真正防止重复渲染，需要保证 props 引用稳定

## 详细讲解
虽然 `config` 里的内容每次看起来都一样，都是 `{ name: "React" }`，但这个对象是在 JSX 里直接创建的，所以每次 `App` 渲染时都会得到一个新的对象引用。React.memo 比较 props 时，看到的是“新对象引用”和“旧对象引用”不相等，因此会认为 props 变化了，进而重新执行子组件。所以点击后仍然会再次输出 `child render React`。
