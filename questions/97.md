# React 输出题 97：函数式更新会改变下一次渲染结果，但不会改变当前回调里的旧值

## 基本信息
- 分类：执行顺序 / 闭包与状态
- 难度：中级
- 标签：函数式更新, 旧闭包, 事件处理
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，这段代码会输出什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  return (
    <button
      onClick={() => {
        setCount((value) => value + 1);
        console.log("after set", count);
      }}
    >
      {count}
    </button>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `render 0`。点击一次后会继续输出 `after set 0 -> render 1`。

## 考察点
- 函数式更新拿到的是最新 state，但当前回调闭包仍然是旧值
- `setState` 之后立刻读 state，读到的不是更新后的结果
- 要区分“下一次渲染会变”与“当前这次回调里已经变了”

## 详细讲解
初次挂载时输出 `render 0`。点击按钮后，虽然 `setCount((value) => value + 1)` 会把下一次渲染的 `count` 更新为 `1`，但紧接着的 `console.log("after set", count)` 读取的还是当前事件回调所属那次渲染的 `count`，也就是 `0`。因此先输出 `after set 0`。等事件处理函数结束后，React 才会进入下一轮渲染，于是再输出 `render 1`。
