# React 输出题 89：在父组件内部定义子组件会让它每次渲染都像新类型

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：高级
- 标签：组件类型, 函数定义, 重新挂载
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，为什么 `Child` 的 state 没有保留下来？控制台输出是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  function Child() {
    const [value] = React.useState(100);
    console.log("child", value);
    return null;
  }

  console.log("app", count);

  return (
    <button onClick={() => setCount(count + 1)}>
      <Child />
    </button>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app 0`、`child 100`。点击一次后会继续输出 `app 1`、`child 100`。

## 考察点
- React 会把组件函数本身当作类型的一部分
- 在父组件内部重新声明子组件，会导致每次渲染都得到新的组件类型
- 组件类型不稳定时，diff 无法复用旧实例

## 详细讲解
第一次渲染时，`App` 执行并输出 `app 0`，随后内部定义的 `Child` 被渲染，输出 `child 100`。点击按钮后，`App` 重新执行，这次会重新创建一个新的 `Child` 函数对象。虽然代码长得一样，但对 React 来说，这已经是一个新的组件类型了。类型一变，旧实例就无法复用，因此新的 `Child` 会重新挂载并再次初始化 state，继续输出 `child 100`。
