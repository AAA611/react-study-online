# React 输出题 68：空依赖 useLayoutEffect 也只会在首次挂载时执行一次

## 基本信息
- 分类：Hooks / useLayoutEffect
- 难度：中级
- 标签：useLayoutEffect, 空依赖数组, props
- 题型：阅读 React 代码并写出输出

## 题目描述
父组件更新 props 后，子组件中的 `useLayoutEffect` 会不会再次执行？

## 代码
```jsx
function Child({ value }) {
  console.log("child render", value);

  React.useLayoutEffect(() => {
    console.log("child layout", value);
  }, []);

  return null;
}

function App() {
  const [value, setValue] = React.useState("A");

  React.useEffect(() => {
    setValue("B");
  }, []);

  return <Child value={value} />;
}

render(<App />);
```

## 参考答案
输出顺序是：`child render A -> child layout A -> child render B`。

## 考察点
- 空依赖的 useLayoutEffect 和空依赖的 useEffect 一样，只在挂载后执行一次
- props 变化会导致组件重新渲染，但不会重新执行空依赖 layout effect
- useLayoutEffect 读到的是首次挂载那一轮的值

## 详细讲解
第一次渲染时，`Child` 先输出 `child render A`。提交后，空依赖的 `useLayoutEffect` 执行一次，于是输出 `child layout A`。父组件的 effect 把 `value` 从 `A` 改成 `B` 后，`Child` 会重新渲染，所以再输出一次 `child render B`。但是由于 `useLayoutEffect` 的依赖数组是空的，它不会因为 props 改变而再次执行。
