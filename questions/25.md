# React 输出题 25：useEffect 中的对象依赖每次渲染都会变化

## 基本信息
- 分类：Hooks / 依赖比较
- 难度：中级
- 标签：依赖数组, 对象引用, effect
- 题型：阅读 React 代码并写出输出

## 题目描述
为什么这个 effect 每次点击都会重新执行？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const options = { step: 1 };

  React.useEffect(() => {
    console.log("effect");
  }, [options]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
首次渲染会打印 `render 0` 和 `effect`。之后每点击一次，都会再次打印对应的 `render n` 和 `effect`。

## 考察点
- 依赖数组比较的是引用而不是内容
- 新的对象字面量就意味着依赖已变化

## 详细讲解
`options` 虽然内容看起来没有变化，但每次渲染都会生成一个新的对象引用。依赖数组比较的是引用是否相同，因此 React 会认为 `options` 每次都变了，于是 effect 在每次点击后都会重新执行。
