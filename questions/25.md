# React Output Question 25: Object dependencies in useEffect change every render

## Basic Info
- Category: Hooks / dependency comparison
- Difficulty: Intermediate
- Tags: dependency array, object reference, effect
- Type: Read React code and write the output

## Description
为什么这个 effect 每次点击都会重新执行？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const options = { step: 1 };

  React.useEffect(() => {
    console.log("effect");
  }, [options]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## Answer
First render prints render 0 and effect. Each click prints render n and effect again.

## Key Points
- dependency arrays compare references
- new object literal means changed dependency

## Detailed Explanation
`options` 虽然内容看起来没有变化，但每次渲染都会生成一个新的对象引用。依赖数组比较的是引用是否相同，因此 React 会认为 `options` 每次都变了，于是 effect 在每次点击后都会重新执行。
