# React 输出题 08：依赖不变时 useMemo 不会重新计算

## 基本信息
- 分类：性能 / useMemo
- 难度：初级
- 标签：useMemo, 依赖项, 记忆化
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，哪些日志会再次出现？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const [text] = React.useState("hello");

  const value = React.useMemo(() => {
    console.log("compute");
    return text.toUpperCase();
  }, [text]);

  console.log("render", count, value);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
首次渲染会打印 `compute` 和 `render 0 HELLO`。点击一次后只会打印 `render 1 HELLO`，不会再次打印 `compute`。

## 考察点
- useMemo 只会在依赖变化时重新计算

## 详细讲解
这里 `useMemo` 的依赖只有 `text`，而 `text` 从头到尾都没有变化，所以缓存结果会被直接复用。点击按钮以后，`count` 变化会让组件重新渲染，因此 `render` 日志会再次出现；但因为依赖没变，`compute` 不会再执行。
