# React Output Question 08: useMemo does not recompute when dependency stays the same

## Basic Info
- Category: Performance / useMemo
- Difficulty: Junior
- Tags: useMemo, dependencies, memoization
- Type: Read React code and write the output

## Description
点击按钮一次后，哪些日志会再次出现？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const [text] = React.useState("hello");

  const value = React.useMemo(() => {
    console.log("compute");
    return text.toUpperCase();
  }, [text]);

  console.log("render", count, value);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## Answer
First render prints compute and render 0 HELLO. After one click, only render 1 HELLO appears.

## Key Points
- useMemo recalculates only when dependencies change

## Detailed Explanation
这里 `useMemo` 的依赖只有 `text`，而 `text` 从头到尾都没有变化，所以缓存结果会被直接复用。点击按钮以后，`count` 变化会让组件重新渲染，因此 `render` 日志会再次出现；但因为依赖没变，`compute` 不会再执行。
