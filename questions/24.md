# React Output Question 24: Calling hooks conditionally breaks hook ordering

## Basic Info
- Category: Hooks rules
- Difficulty: Intermediate
- Tags: hooks rules, conditional hook, error
- Type: Read React code and write the output

## Description
第二次渲染后会发生什么？

## Code
```jsx
function App() {
  const [flag, setFlag] = React.useState(true);

  if (flag) {
    React.useEffect(() => {
      console.log("effect");
    }, []);
  }

  React.useEffect(() => {
    setFlag(false);
  }, []);

  return null;
}

render(<App />);
```

## Answer
The second render breaks hook order and React throws a hooks error such as Rendered fewer hooks than expected.

## Key Points
- hooks must be called unconditionally in the same order

## Detailed Explanation
第一次渲染时，`if` 分支成立，所以里面的 `useEffect` 被调用了。第二次渲染时 `flag` 变成 `false`，这个 hook 不再执行，导致前后两次渲染的 hook 调用顺序不一致。React 依赖固定顺序来管理 hook，因此这里会直接报错。
