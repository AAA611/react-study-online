# React 输出题 24：条件调用 Hook 会破坏 Hook 顺序

## 基本信息
- 分类：Hooks 规则
- 难度：中级
- 标签：Hook 规则, 条件 Hook, 错误
- 题型：阅读 React 代码并写出输出

## 题目描述
第二次渲染后会发生什么？

## 代码
```jsx
function App() {
  const [flag, setFlag] = React.useState(true);

  if (flag) {
    React.useEffect(() => {
      console.log("effect");
    }, []);
  }

  React.useEffect(() => {
    setFlag(false);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
第二次渲染会破坏 Hook 的调用顺序，React 会直接报错，常见报错类似 `Rendered fewer hooks than expected`。

## 考察点
- Hook 必须以固定顺序、无条件地调用

## 详细讲解
第一次渲染时，`if` 分支成立，所以里面的 `useEffect` 被调用了。第二次渲染时 `flag` 变成 `false`，这个 hook 不再执行，导致前后两次渲染的 hook 调用顺序不一致。React 依赖固定顺序来管理 hook，因此这里会直接报错。
