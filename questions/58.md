# React 输出题 58：列表头部插入元素且使用稳定 `key` 时状态不会错位

## 基本信息
- 分类：列表 / key
- 难度：中级
- 标签：diff, key, 列表复用
- 题型：阅读 React 代码并写出输出

## 题目描述
和上一题几乎一样，但这次用 `label` 作为稳定 `key`。控制台输出会变成什么？

## 代码
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["B", "C"]);

  React.useEffect(() => {
    setList(["A", "B", "C"]);
  }, []);

  return list.map((label) => (
    <Item key={label} label={label} />
  ));
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `B B`、`C C`，更新后再打印 `A A`、`B B`、`C C`。

## 考察点
- 稳定 `key` 能帮助 React 正确识别每个列表项
- 头部插入新元素时，旧项不会因为位置变化而状态串位
- 新增项会新建实例，旧项会按身份复用

## 详细讲解
第一次渲染时输出 `B B`、`C C`。更新后，列表前面多了一个 `A`，但每项都使用自身内容作为稳定 `key`。React diff 时会认出旧的 `B` 还是旧的 `B`，旧的 `C` 还是旧的 `C`，只需要额外创建一个新的 `A`。所以第二轮输出是 `A A`、`B B`、`C C`，不存在上一题那种位置错位导致的状态串线问题。
