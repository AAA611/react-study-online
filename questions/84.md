# React 输出题 84：同一个 key 如果对应的组件类型变了，依然会重新挂载

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：key, 组件类型, 重新挂载
- 题型：阅读 React 代码并写出输出

## 题目描述
下面两个位置的 key 都保持不变，但 key 对应的组件类型交换了。控制台会输出什么？

## 代码
```jsx
function Foo() {
  const [value] = React.useState(1);
  console.log("Foo", value);
  return null;
}

function Bar() {
  const [value] = React.useState(2);
  console.log("Bar", value);
  return null;
}

function App() {
  const [swap, setSwap] = React.useState(false);

  React.useEffect(() => {
    setSwap(true);
  }, []);

  return swap
    ? [<Foo key="x" />, <Bar key="y" />]
    : [<Bar key="x" />, <Foo key="y" />];
}

render(<App />);
```

## 参考答案
输出顺序是：`Bar 2 -> Foo 1 -> Foo 1 -> Bar 2`。

## 考察点
- key 只能帮助 React 定位节点，不能跨组件类型强行复用
- React diff 时最终还是要比较组件类型
- “key 一样”不代表“实例一定会被保留”

## 详细讲解
第一次渲染时，key 为 `"x"` 的节点是 `Bar`，所以输出 `Bar 2`；key 为 `"y"` 的节点是 `Foo`，所以输出 `Foo 1`。更新后，key `"x"` 对应的节点变成了 `Foo`，key `"y"` 对应的节点变成了 `Bar`。虽然 key 没变，但它们各自绑定的组件类型变了。React 在匹配到相同 key 后，仍然会检查类型，发现类型不同就会把旧实例卸载、新实例重新挂载，因此后面会重新输出 `Foo 1` 和 `Bar 2`。
