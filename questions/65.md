# React 输出题 65：同一轮中多个 state 更新会影响另一个 useEffect 的下一次执行

## 基本信息
- 分类：Hooks / useEffect
- 难度：中级
- 标签：useEffect, 批处理, 多个状态
- 题型：阅读 React 代码并写出输出

## 题目描述
下面两个 effect 配合时，控制台输出顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const [ready, setReady] = React.useState(false);

  console.log("render", count, ready);

  React.useEffect(() => {
    console.log("effect A");
    setCount(1);
    setReady(true);
  }, []);

  React.useEffect(() => {
    console.log("effect B", count, ready);
  }, [count, ready]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 false -> effect A -> effect B 0 false -> render 1 true -> effect B 1 true`。

## 考察点
- 多个 effect 会按声明顺序执行
- 同一轮中的多个 state 更新通常会被批处理
- 依赖型 effect 会在下一轮渲染完成后拿到最新状态

## 详细讲解
首次渲染时输出 `render 0 false`。提交后，先执行第一个 effect，也就是 `effect A`，它会把 `count` 和 `ready` 一起更新。接着同一轮中第二个 effect 也会执行，但它拿到的仍然是这一轮渲染对应的值，所以输出 `effect B 0 false`。React 随后把前面的两个状态更新合并到下一次渲染中，因此重新渲染时输出 `render 1 true`。这一轮提交后，第二个 effect 因依赖变化而再次执行，于是输出 `effect B 1 true`。
