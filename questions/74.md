# React 输出题 74：useMemo 可以稳定对象 props，从而减少子组件重复渲染

## 基本信息
- 分类：性能优化 / useMemo
- 难度：中级
- 标签：useMemo, React.memo, 对象稳定
- 题型：阅读 React 代码并写出输出

## 题目描述
下面这段代码里，点击按钮后子组件还会重新渲染吗？控制台输出是什么？

## 代码
```jsx
const Child = React.memo(function Child({ config }) {
  console.log("child render", config.name);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  const config = React.useMemo(() => ({ name: "React" }), []);

  console.log("app render", count);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <Child config={config} />
    </>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render React`。点击一次后只会输出 `app render 1`。

## 考察点
- useMemo 可以缓存对象引用
- 稳定的对象 props 配合 React.memo 才能真正减少重复渲染
- 优化重点不只是“值一样”，还包括“引用要稳定”

## 详细讲解
第一次渲染时，`config` 会被创建一次，并传给 `Child`，所以输出 `app render 0` 和 `child render React`。之后点击按钮，`App` 会重新渲染并输出 `app render 1`。但是 `config` 是通过 `useMemo(..., [])` 创建的，依赖没有变化，因此它会继续复用同一个对象引用。`Child` 又被 `React.memo` 包裹，所以这次浅比较会发现 `config` 没变，于是子组件不会重新执行。
