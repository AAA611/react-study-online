# React 输出题 22：useState 的惰性初始化只会执行一次

## 基本信息
- 分类：Hooks / useState
- 难度：初级
- 标签：惰性初始化, 首次渲染, useState
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮后，初始化函数还会不会再次执行？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(() => {
    console.log("init");
    return 0;
  });

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `init`，再打印 `render 0`。点击一次后只会打印 `render 1`。

## 考察点
- 惰性初始化只会在挂载时执行一次

## 详细讲解
`useState` 的函数初始化器只会在首次挂载时执行一次，用来计算初始 state。后面的重新渲染都会直接复用 React 保存的 state，因此再次点击时不会重新打印 `init`。
