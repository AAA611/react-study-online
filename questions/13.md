# React Output Question 13: Index key can cause state mismatch after reordering

## Basic Info
- Category: Lists / key
- Difficulty: Intermediate
- Tags: key, list, state reuse
- Type: Read React code and write the output

## Description
列表从 `A、B` 变成 `B、A` 之后，控制台会打印什么？

## Code
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["A", "B"]);

  React.useEffect(() => {
    setList(["B", "A"]);
  }, []);

  return list.map((item, index) => <Item key={index} label={item} />);
}

render(<App />);
```

## Answer
Logs: A A, B B, then B A, A B.

## Key Points
- index keys reuse instances by position
- state can move to the wrong item after reorder

## Detailed Explanation
当 `key` 使用 `index` 时，React 只能按位置复用组件实例，而不是按元素真实身份复用。顺序调整后，第一个位置还是复用旧的第一个实例，第二个位置复用旧的第二个实例，所以 state 会跟着位置走，最终出现 `B A`、`A B` 这种错位结果。
