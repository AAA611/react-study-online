# React 输出题 80：useMemo 可以避免无关状态变化时重复执行昂贵计算

## 基本信息
- 分类：性能优化 / useMemo
- 难度：中级
- 标签：useMemo, 昂贵计算, 重复计算
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮后，为什么不会再次打印 `compute`？控制台输出是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const [query] = React.useState("React");

  const result = React.useMemo(() => {
    console.log("compute", query);
    return query.toUpperCase();
  }, [query]);

  console.log("render", count, result);

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
首次渲染会输出 `compute React`、`render 0 REACT`。点击一次后只会输出 `render 1 REACT`。

## 考察点
- useMemo 会缓存计算结果
- 依赖不变时，无关状态更新不会让记忆化计算重复执行
- 性能优化不只是减少组件渲染，也包括减少重复计算

## 详细讲解
第一次渲染时，`useMemo` 会执行回调，所以先输出 `compute React`，并得到结果 `REACT`。随后组件继续渲染，输出 `render 0 REACT`。点击按钮后，变化的是 `count`，而 `query` 没有变，所以 `useMemo` 会直接复用上一次缓存的结果，不会再次执行回调，因此不会再打印 `compute React`。这时组件还是会重新渲染，所以只会输出 `render 1 REACT`。
