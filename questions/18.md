# React 输出题 18：useLayoutEffect 会先于 useEffect 执行

## 基本信息
- 分类：Hooks / 执行时机
- 难度：中级
- 标签：useLayoutEffect, useEffect, 执行时机
- 题型：阅读 React 代码并写出输出

## 题目描述
请写出完整的输出顺序。

## 代码
```jsx
function App() {
  console.log("render");

  React.useLayoutEffect(() => {
    console.log("layout");
  }, []);

  React.useEffect(() => {
    console.log("effect");
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render -> layout -> effect`。

## 考察点
- layout effect 的执行时机早于普通 effect
- 不管是哪一种 effect，渲染都会先发生

## 详细讲解
组件函数先执行，所以 `render` 最先输出。提交 DOM 后，`useLayoutEffect` 会先同步执行，因此输出 `layout`。普通的 `useEffect` 会在更晚的时机执行，所以最后才是 `effect`。
