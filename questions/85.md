# React 输出题 85：带 key 的 Fragment 也能作为一个可复用的分组单位

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：Fragment, key, 分组复用
- 题型：阅读 React 代码并写出输出

## 题目描述
两个 `Row` 都包在带 key 的 `Fragment` 里，更新后只是调换了分组顺序。控制台输出是什么？

## 代码
```jsx
function Row({ name }) {
  const [value] = React.useState(name);
  console.log("row", name, value);
  return null;
}

function App() {
  const [swap, setSwap] = React.useState(false);

  React.useEffect(() => {
    setSwap(true);
  }, []);

  return swap
    ? [
        <React.Fragment key="right">
          <Row name="B" />
        </React.Fragment>,
        <React.Fragment key="left">
          <Row name="A" />
        </React.Fragment>
      ]
    : [
        <React.Fragment key="left">
          <Row name="A" />
        </React.Fragment>,
        <React.Fragment key="right">
          <Row name="B" />
        </React.Fragment>
      ];
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `row A A`、`row B B`，更新后再打印 `row B B`、`row A A`。

## 考察点
- Fragment 只要带了稳定 key，也可以参与列表 diff
- React 可以按分组身份复用整组内容
- 组件顺序变化不一定意味着状态错位

## 详细讲解
第一次渲染时，依次输出 `row A A` 和 `row B B`。更新后，左右两个分组只是交换了顺序，但每个 `Fragment` 都有稳定的 key，所以 React 能知道“左边那组还是 left”“右边那组还是 right”。因此内部的 `Row` 实例会跟着各自的分组一起移动，而不是按位置乱复用。第二轮输出 `row B B`、`row A A`，说明顺序变了，但各自的 state 仍然保留正确。
