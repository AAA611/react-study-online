# React 输出题 78：单个 Context 值对象变化会让不相关消费者也重新渲染

## 基本信息
- 分类：性能优化 / Context
- 难度：中级
- 标签：Context, 重复渲染, Provider
- 题型：阅读 React 代码并写出输出

## 题目描述
`ThemeText` 只读取 `theme`，为什么点击后它还是会重新渲染？

## 代码
```jsx
const AppContext = React.createContext(null);

function ThemeText() {
  const value = React.useContext(AppContext);
  console.log("theme render", value.theme);
  return null;
}

function CounterButton() {
  const value = React.useContext(AppContext);
  return <button onClick={() => value.setCount((n) => n + 1)}>{value.count}</button>;
}

function App() {
  const [count, setCount] = React.useState(0);

  return (
    <AppContext.Provider value={{ theme: "dark", count, setCount }}>
      <ThemeText />
      <CounterButton />
    </AppContext.Provider>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `theme render dark`。点击一次后会再次输出 `theme render dark`。

## 考察点
- Context 消费者会跟随 Provider value 的变化重新渲染
- 即使只用到对象里的某一个字段，也无法自动跳过更新
- Provider 的对象字面量每次都会创建新引用

## 详细讲解
第一次渲染时，`ThemeText` 从 Context 里读取 `theme`，输出 `theme render dark`。点击按钮后，`count` 变化，`App` 重新渲染，Provider 的 `value` 也会重新创建成一个新对象 `{ theme: "dark", count, setCount }`。对于 Context 来说，只要 `value` 变了，订阅它的消费者就会重新渲染，所以 `ThemeText` 虽然只关心 `theme`，也还是会再次执行并输出 `theme render dark`。这类场景常见的优化方式是拆分 Context，或者把不同频率的数据拆开提供。
