# React 输出题 52：同一位置但组件类型不同会直接替换

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, 组件替换, 节点类型
- 题型：阅读 React 代码并写出输出

## 题目描述
同一位置先渲染 `Foo`，后渲染 `Bar`。React 会怎么处理？控制台输出是什么？

## 代码
```jsx
function Foo() {
  const [value] = React.useState(1);
  console.log("Foo", value);
  return null;
}

function Bar() {
  const [value] = React.useState(2);
  console.log("Bar", value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag ? <Bar /> : <Foo />;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `Foo 1`，再打印 `Bar 2`。

## 考察点
- React diff 时会先比较节点类型
- 同一位置上类型不同的组件不会被复用
- 类型变化会导致旧子树被卸载、新子树重新挂载

## 详细讲解
第一次渲染时，`App` 返回的是 `<Foo />`，所以会输出 `Foo 1`。effect 执行后，`flag` 变成 `true`，同一位置上的节点从 `Foo` 变成了 `Bar`。React 在 diff 时发现这两个节点的类型不同，就不会复用旧实例，而是直接把旧的 `Foo` 子树替换掉，再挂载新的 `Bar` 子树。因此第二次输出是 `Bar 2`，而不是沿用 `Foo` 内部的状态。
