# React 输出题 19：组件内部创建的对象每次渲染都是新的

## 基本信息
- 分类：渲染模型 / 引用
- 难度：初级
- 标签：对象引用, 重渲染, 身份标识
- 题型：阅读 React 代码并写出输出

## 题目描述
点击一次后，`same?` 比较结果是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const obj = { value: 1 };
  const prevRef = React.useRef();

  console.log("same?", prevRef.current === obj);
  prevRef.current = obj;

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
首次渲染会打印 `same? false`。点击一次后仍然会打印 `same? false`。

## 考察点
- 每次重渲染时，组件函数体都会重新执行
- 对象字面量每次渲染都会重新创建

## 详细讲解
函数组件每次重新渲染都会重新执行整个函数体，因此对象字面量 `obj` 也会被重新创建。`prevRef.current` 存的是上一次渲染中的对象，所以无论首次还是更新后，比较结果都会是 `false`。
