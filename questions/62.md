# React 输出题 62：带 cleanup 的 useEffect 在依赖变化时先清理再执行新 effect

## 基本信息
- 分类：Hooks / useEffect
- 难度：中级
- 标签：useEffect, cleanup, 依赖变化
- 题型：阅读 React 代码并写出输出

## 题目描述
`count` 从 `0` 变成 `1` 时，旧 effect 和新 effect 的执行顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect", count);

    if (count === 0) {
      setCount(1);
    }

    return () => {
      console.log("cleanup", count);
    };
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> effect 0 -> render 1 -> cleanup 0 -> effect 1`。

## 考察点
- 依赖变化时，旧 effect 的 cleanup 会先执行
- cleanup 拿到的是上一次渲染闭包中的值
- 新 effect 会在旧 cleanup 完成后执行

## 详细讲解
首次渲染输出 `render 0`，随后 effect 执行，输出 `effect 0`，并把 `count` 更新成 `1`。组件重新渲染时先输出 `render 1`。因为依赖从 `0` 变成了 `1`，React 会先清理旧的 effect，所以输出 `cleanup 0`。注意这里 cleanup 里读到的还是旧闭包里的 `count`，因此是 `0`。清理完成后，再执行新的 effect，于是输出 `effect 1`。
