# React 输出题 69：多个 useLayoutEffect 和 useEffect 都按声明顺序执行

## 基本信息
- 分类：Hooks / 执行顺序
- 难度：中级
- 标签：useLayoutEffect, useEffect, 声明顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
同一个组件里声明了两个 `useLayoutEffect` 和两个 `useEffect`，首次挂载时输出顺序是什么？

## 代码
```jsx
function App() {
  console.log("render");

  React.useLayoutEffect(() => {
    console.log("layout 1");
  }, []);

  React.useLayoutEffect(() => {
    console.log("layout 2");
  }, []);

  React.useEffect(() => {
    console.log("effect 1");
  }, []);

  React.useEffect(() => {
    console.log("effect 2");
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render -> layout 1 -> layout 2 -> effect 1 -> effect 2`。

## 考察点
- 同类 Hook 会按声明顺序执行
- useLayoutEffect 整体先于 useEffect 执行
- 执行顺序既受 Hook 类型影响，也受代码声明顺序影响

## 详细讲解
组件函数体先运行，所以最先输出 `render`。提交后，React 会先处理所有的 `useLayoutEffect`，并且顺序与代码声明顺序一致，因此先输出 `layout 1`，再输出 `layout 2`。之后才轮到 `useEffect`，同样按声明顺序执行，所以接着输出 `effect 1`、`effect 2`。
