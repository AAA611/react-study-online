# React 输出题 05：cleanup 会先于下一次 effect 执行

## 基本信息
- 分类：Hooks / 清理函数
- 难度：中级
- 标签：清理函数, useEffect, 更新顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
当 `flag` 从 `false` 变成 `true` 时，日志顺序是什么？

## 代码
```jsx
function App() {
  const [flag, setFlag] = React.useState(false);

  console.log("render", flag);

  React.useEffect(() => {
    console.log("effect", flag);
    return () => console.log("cleanup", flag);
  }, [flag]);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render false -> effect false -> render true -> cleanup false -> effect true`。

## 考察点
- cleanup 读到的是上一次渲染闭包里的值
- 依赖变化时，cleanup 会先于下一次 effect 执行

## 详细讲解
首次渲染先输出 `render false`。提交完成后，第一个 effect 输出 `effect false`，第二个 effect 把 `flag` 更新为 `true`。组件重新渲染后会先得到 `render true`，然后在提交阶段先清理上一次依赖旧值的 effect，所以输出 `cleanup false`，最后再执行新的 effect，输出 `effect true`。注意 cleanup 里读取到的仍然是旧闭包中的 `false`。
