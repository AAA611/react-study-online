# React Output Question 05: Cleanup runs before the next effect

## Basic Info
- Category: Hooks / cleanup
- Difficulty: Intermediate
- Tags: cleanup, useEffect, update order
- Type: Read React code and write the output

## Description
当 `flag` 从 `false` 变成 `true` 时，日志顺序是什么？

## Code
```jsx
function App() {
  const [flag, setFlag] = React.useState(false);

  console.log("render", flag);

  React.useEffect(() => {
    console.log("effect", flag);
    return () => console.log("cleanup", flag);
  }, [flag]);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return null;
}

render(<App />);
```

## Answer
Output: render false -> effect false -> render true -> cleanup false -> effect true.

## Key Points
- cleanup sees previous closure values
- cleanup runs before the next effect with changed deps

## Detailed Explanation
首次渲染先输出 `render false`。提交完成后，第一个 effect 输出 `effect false`，第二个 effect 把 `flag` 更新为 `true`。组件重新渲染后会先得到 `render true`，然后在提交阶段先清理上一次依赖旧值的 effect，所以输出 `cleanup false`，最后再执行新的 effect，输出 `effect true`。注意 cleanup 里读取到的仍然是旧闭包中的 `false`。
