# React Output Question 04: Effect dependency change order

## Basic Info
- Category: Hooks / useEffect
- Difficulty: Junior
- Tags: useEffect, dependency array, order
- Type: Read React code and write the output

## Description
请写出组件从首次挂载到更新完成的完整输出顺序。

## Code
```jsx
function App() {
  const [value, setValue] = React.useState(0);

  console.log("render", value);

  React.useEffect(() => {
    console.log("effect", value);
    if (value === 0) setValue(1);
  }, [value]);

  return null;
}

render(<App />);
```

## Answer
Output order: render 0 -> effect 0 -> render 1 -> effect 1.

## Key Points
- render happens before effect
- changing dependency reruns the effect

## Detailed Explanation
首次渲染时 `value` 为 `0`，所以先输出 `render 0`。提交完成后 `effect` 执行，输出 `effect 0`，并因为 `value === 0` 而触发 `setValue(1)`。接着组件重新渲染，输出 `render 1`。由于依赖项 `value` 发生了变化，`effect` 会再次执行，因此最后输出 `effect 1`。
