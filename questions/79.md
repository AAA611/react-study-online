# React 输出题 79：把状态下沉到子组件内部，可以减少父组件区域的重复渲染

## 基本信息
- 分类：性能优化 / 状态下沉
- 难度：中级
- 标签：状态下沉, 重复渲染, 组件边界
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮后，为什么父组件里那些静态区域不会再次执行？控制台输出是什么？

## 代码
```jsx
function Counter() {
  const [count, setCount] = React.useState(0);
  console.log("counter render", count);

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}

function App() {
  console.log("app render");

  return (
    <>
      <h1>React Study</h1>
      <Counter />
    </>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render`、`counter render 0`。点击一次后只会输出 `counter render 1`。

## 考察点
- 谁的 state 更新，谁所在的组件边界就会重新执行
- 把频繁变化的状态下沉到更小的组件里，有助于减少无关区域渲染
- 性能优化不只靠 memo，也可以靠合理拆分组件

## 详细讲解
第一次渲染时，`App` 和 `Counter` 都会执行，所以输出 `app render` 和 `counter render 0`。点击按钮后，发生变化的是 `Counter` 自己内部的 state，而不是 `App` 的 state。因此重新渲染的边界从 `Counter` 开始，`App` 本身不会再次执行，于是只输出 `counter render 1`。这说明把频繁变化的状态放进更小的子组件中，本身就是一种很有效的性能优化方式。
