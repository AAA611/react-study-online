# React 输出题 93：setTimeout 回调会晚于本轮 effect 执行

## 基本信息
- 分类：执行顺序 / effect 与定时器
- 难度：中级
- 标签：useEffect, setTimeout, 异步顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
首次挂载后，这段代码里同步渲染、effect 和定时器回调的顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect", count);

    setTimeout(() => {
      console.log("timeout", count);
      setCount((value) => value + 1);
    }, 0);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> effect 0 -> timeout 0 -> render 1`。

## 考察点
- effect 会在首次提交后执行
- `setTimeout` 回调属于更晚的异步阶段
- 定时器回调读到的仍然是创建它时的闭包值

## 详细讲解
第一次渲染时先输出 `render 0`。提交完成后，`useEffect` 执行，所以输出 `effect 0`。effect 里又安排了一个 `setTimeout`，它不会立刻执行，而是等到更晚的异步阶段才运行，于是输出 `timeout 0`。这里打印的仍然是首次 effect 创建时闭包里的 `count`，所以是 `0`。随后调用 `setCount`，React 再进行一次新的渲染，因此最后输出 `render 1`。
