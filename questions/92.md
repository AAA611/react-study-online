# React 输出题 92：事件、微任务和下一次渲染之间的先后关系

## 基本信息
- 分类：执行顺序 / 事件与异步
- 难度：高级
- 标签：Promise, 微任务, setState
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，事件处理函数、`Promise.then` 和重渲染之间的顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  return (
    <button
      onClick={() => {
        console.log("click", count);

        Promise.resolve().then(() => {
          console.log("promise", count);
          setCount((value) => value + 1);
        });
      }}
    >
      {count}
    </button>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `render 0`。点击一次后会继续输出 `click 0 -> promise 0 -> render 1`。

## 考察点
- 微任务会在当前同步代码结束后执行
- `Promise.then` 里的闭包仍然读取旧渲染快照
- 微任务中的状态更新会触发下一次渲染

## 详细讲解
初次挂载时先输出 `render 0`。点击按钮后，事件处理函数同步执行，所以先输出 `click 0`。`Promise.then` 会被放进微任务队列，等当前同步任务结束后再执行，因此接着输出 `promise 0`。这里读到的 `count` 依然是点击发生时那一轮渲染里的值 `0`。随后 `setCount` 触发新的渲染，所以最后输出 `render 1`。
