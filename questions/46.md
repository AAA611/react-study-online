# React Output Question 46: Promise callbacks are also batched in modern React

## Basic Info
- Category: Async updates / batching
- Difficulty: Intermediate
- Tags: Promise, automatic batching, state
- Type: Read React code and write the output

## Description
`Promise.then` 里连续两次函数式更新，最终会输出什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  console.log("render", count);

  React.useEffect(() => {
    Promise.resolve().then(() => {
      setCount(c => c + 1);
      setCount(c => c + 1);
    });
  }, []);

  return null;
}

render(<App />);
```

## Answer
Usually it logs render 0, then render 2.

## Key Points
- automatic batching also covers many async cases in React 18+
- functional updates still accumulate correctly

## Detailed Explanation
两次更新都发生在同一个 `Promise` 微任务里，并且都使用了函数式更新，因此结果会从 `0` 连续加到 `2`。现代 React 会把这类异步场景中的更新自动批处理，所以通常只会额外渲染一次，直接输出 `render 2`。
