# React 输出题 46：Promise 回调在现代 React 中也会被批处理

## 基本信息
- 分类：异步更新 / 批处理
- 难度：中级
- 标签：Promise, 自动批处理, 状态
- 题型：阅读 React 代码并写出输出

## 题目描述
`Promise.then` 里连续两次函数式更新，最终会输出什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  console.log("render", count);

  React.useEffect(() => {
    Promise.resolve().then(() => {
      setCount(c => c + 1);
      setCount(c => c + 1);
    });
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
通常会看到 `render 0`，然后直接打印 `render 2`。

## 考察点
- React 18+ 的自动批处理也覆盖了很多异步场景
- 函数式更新在这种场景里仍然可以正确累加

## 详细讲解
两次更新都发生在同一个 `Promise` 微任务里，并且都使用了函数式更新，因此结果会从 `0` 连续加到 `2`。现代 React 会把这类异步场景中的更新自动批处理，所以通常只会额外渲染一次，直接输出 `render 2`。
