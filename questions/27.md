# React 输出题 27：Provider 的 value 变化时消费者会重渲染

## 基本信息
- 分类：Context / 上下文
- 难度：中级
- 标签：上下文, Provider, Consumer
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮前后，子组件分别会输出什么？

## 代码
```jsx
const CountContext = React.createContext(0);

function Child() {
  const value = React.useContext(CountContext);
  console.log("child", value);
  return null;
}

function App() {
  const [count, setCount] = React.useState(0);

  return (
    <CountContext.Provider value={count}>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Child />
    </CountContext.Provider>
  );
}

render(<App />);
```

## 参考答案
首次挂载时会打印 `child 0`；点击一次后会再次打印 `child 1`。

## 考察点
- Consumer 会订阅 Provider value 的变化

## 详细讲解
`Child` 通过 `useContext` 订阅了 `CountContext`。当 Provider 的 `value` 从 `0` 变成 `1` 时，React 会通知消费者更新，因此子组件会重新渲染并输出 `child 1`。
