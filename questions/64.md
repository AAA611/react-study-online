# React 输出题 64：空依赖 useEffect 不会因为 props 变化而重新执行

## 基本信息
- 分类：Hooks / useEffect
- 难度：初级
- 标签：useEffect, 空依赖数组, props
- 题型：阅读 React 代码并写出输出

## 题目描述
父组件把 `label` 从 `A` 改成 `B` 后，子组件中的 effect 会不会再次执行？

## 代码
```jsx
function Child({ label }) {
  console.log("child render", label);

  React.useEffect(() => {
    console.log("child effect", label);
  }, []);

  return null;
}

function App() {
  const [label, setLabel] = React.useState("A");

  React.useEffect(() => {
    setLabel("B");
  }, []);

  return <Child label={label} />;
}

render(<App />);
```

## 参考答案
输出顺序是：`child render A -> child effect A -> child render B`。

## 考察点
- 空依赖数组的 effect 只会在首次挂载后执行一次
- props 变化会让组件重新渲染，但不会让空依赖 effect 重新执行
- effect 内部读到的是挂载时那一轮渲染的值

## 详细讲解
第一次渲染时，`Child` 先输出 `child render A`。提交完成后，空依赖的 effect 执行一次，因此输出 `child effect A`。父组件自己的 effect 把 `label` 改成 `B` 后，`Child` 会因为 props 更新而重新渲染，所以又输出一次 `child render B`。但是由于 `Child` 里的 effect 依赖数组是空的，它不会因为 props 改变而再次执行，所以不会再输出 `child effect B`。
