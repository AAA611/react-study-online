# React Output Question 34: dispatch in useReducer does not update the current closure immediately

## Basic Info
- Category: Hooks / useReducer
- Difficulty: Junior
- Tags: useReducer, dispatch, closure
- Type: Read React code and write the output

## Description
点击按钮一次时，控制台会输出什么？

## Code
```jsx
function reducer(state, action) {
  if (action.type === "add") return state + 1;
  return state;
}

function App() {
  const [count, dispatch] = React.useReducer(reducer, 0);

  const handleClick = () => {
    dispatch({ type: "add" });
    console.log("click", count);
  };

  console.log("render", count);
  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## Answer
Output: render 0, then click 0, then render 1.

## Key Points
- dispatch schedules a new render
- the current event callback still sees the old state snapshot

## Detailed Explanation
`dispatch` 和 `setState` 一样，都会触发下一次渲染，但不会立刻改掉当前事件回调中的 `count` 变量。所以点击时先打印 `click 0`，之后组件才会重新渲染并输出 `render 1`。
