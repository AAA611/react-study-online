# React 输出题 34：useReducer 的 dispatch 不会立刻更新当前闭包

## 基本信息
- 分类：Hooks / useReducer
- 难度：初级
- 标签：useReducer, dispatch, 闭包
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次时，控制台会输出什么？

## 代码
```jsx
function reducer(state, action) {
  if (action.type === "add") return state + 1;
  return state;
}

function App() {
  const [count, dispatch] = React.useReducer(reducer, 0);

  const handleClick = () => {
    dispatch({ type: "add" });
    console.log("click", count);
  };

  console.log("render", count);
  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> click 0 -> render 1`。

## 考察点
- dispatch 会安排一次新的渲染
- 当前这个事件回调里看到的仍然是旧状态快照

## 详细讲解
`dispatch` 和 `setState` 一样，都会触发下一次渲染，但不会立刻改掉当前事件回调中的 `count` 变量。所以点击时先打印 `click 0`，之后组件才会重新渲染并输出 `render 1`。
