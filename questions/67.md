# React 输出题 67：先执行 useLayoutEffect，再执行 useEffect 触发的下一轮更新

## 基本信息
- 分类：Hooks / useLayoutEffect
- 难度：中级
- 标签：useLayoutEffect, useEffect, 更新顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
下面这段代码里，首次挂载后谁先执行？状态更新后又会发生什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useLayoutEffect(() => {
    console.log("layout", count);
  }, [count]);

  React.useEffect(() => {
    console.log("effect", count);

    if (count === 0) {
      setCount(1);
    }
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> layout 0 -> effect 0 -> render 1 -> layout 1 -> effect 1`。

## 考察点
- 每一轮渲染中，useLayoutEffect 都先于 useEffect 执行
- useEffect 中触发的 state 更新会开启下一轮渲染
- 依赖变化后，两种 effect 都会再次执行

## 详细讲解
首次渲染先输出 `render 0`。提交后，先执行 `useLayoutEffect`，输出 `layout 0`，再执行 `useEffect`，输出 `effect 0`。因为 `count === 0`，这个 effect 会把状态更新成 `1`。React 随后进入下一轮渲染，输出 `render 1`。提交后继续遵循相同顺序，先输出 `layout 1`，再输出 `effect 1`。由于这时条件不再满足，不会继续更新。
