# React 输出题 96：一个组件里的多个 effect 会按声明顺序依次执行

## 基本信息
- 分类：执行顺序 / 多个 effect
- 难度：中级
- 标签：useEffect, 声明顺序, 多个副作用
- 题型：阅读 React 代码并写出输出

## 题目描述
首次挂载时，这两个 effect 的输出顺序是什么？

## 代码
```jsx
function App() {
  console.log("render");

  React.useEffect(() => {
    console.log("effect 1");
  }, []);

  React.useEffect(() => {
    console.log("effect 2");
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render -> effect 1 -> effect 2`。

## 考察点
- 同一个组件里的多个 effect 会按代码声明顺序执行
- render 阶段总是先于 effect 阶段
- 执行顺序既和 Hook 类型有关，也和声明顺序有关

## 详细讲解
组件函数体先运行，所以最先输出 `render`。提交完成后，React 会开始执行这个组件里声明的 effect，并且顺序就是代码里写出来的顺序。因此先输出 `effect 1`，再输出 `effect 2`。这也是为什么 effect 的先后关系通常可以直接从源码顺序判断出来。
