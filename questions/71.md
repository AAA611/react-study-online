# React 输出题 71：父组件更新时普通子组件也会跟着重新渲染

## 基本信息
- 分类：性能优化 / 重复渲染
- 难度：初级
- 标签：渲染, 父子组件, 重复渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
点击一次按钮后，为什么子组件也会再次执行？控制台输出是什么？

## 代码
```jsx
function Child() {
  console.log("child render");
  return null;
}

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <Child />
    </>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render`。点击一次后会继续输出 `app render 1`、`child render`。

## 考察点
- 父组件重渲染时，普通子组件默认也会重新执行
- React 不会自动帮普通函数组件跳过重复渲染
- 防止无意义重渲染通常需要额外的优化手段

## 详细讲解
第一次渲染时，`App` 和 `Child` 都会执行，所以依次输出 `app render 0` 和 `child render`。点击按钮后，`count` 变成 `1`，父组件 `App` 重新渲染。因为 `Child` 只是一个普通函数组件，没有做任何记忆化处理，所以它也会跟着再次执行，于是又输出一次 `child render`。这也是 React 性能优化里最基础的一点：父组件更新时，子组件并不会天然被“缓存”。
