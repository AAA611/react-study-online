# React Output Question 21: Parent state update rerenders normal children too

## Basic Info
- Category: Render model / parent child
- Difficulty: Junior
- Tags: parent child, rerender, normal component
- Type: Read React code and write the output

## Description
没有 `React.memo` 时，点击按钮后子组件会发生什么？

## Code
```jsx
function Child() {
  console.log("child");
  return null;
}

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Child />
    </div>
  );
}

render(<App />);
```

## Answer
Output: app 0, child, then after one click app 1, child.

## Key Points
- normal children rerender with their parent by default

## Detailed Explanation
父组件状态变化后会重新渲染，而普通函数子组件默认也会随着父组件重新执行一次。因为这里没有用 `React.memo` 之类的优化，所以点击按钮后 `child` 日志也会再次出现。
