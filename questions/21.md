# React 输出题 21：父组件状态更新也会让普通子组件重渲染

## 基本信息
- 分类：渲染模型 / 父子组件
- 难度：初级
- 标签：父子组件, 重渲染, 普通组件
- 题型：阅读 React 代码并写出输出

## 题目描述
没有 `React.memo` 时，点击按钮后子组件会发生什么？

## 代码
```jsx
function Child() {
  console.log("child");
  return null;
}

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <Child />
    </div>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：首次渲染打印 `app 0`、`child`；点击一次后打印 `app 1`、`child`。

## 考察点
- 默认情况下，普通子组件会跟着父组件一起重渲染

## 详细讲解
父组件状态变化后会重新渲染，而普通函数子组件默认也会随着父组件重新执行一次。因为这里没有用 `React.memo` 之类的优化，所以点击按钮后 `child` 日志也会再次出现。
