# React 输出题 51：同一位置的同类型组件会被复用

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, 组件复用, 同类型节点
- 题型：阅读 React 代码并写出输出

## 题目描述
父组件更新后，子组件虽然换了一个 `label`，但还是同一个组件类型、同一个位置。控制台会输出什么？

## 代码
```jsx
function Child({ label }) {
  const [value] = React.useState(label === "A" ? 1 : 999);
  console.log("child", label, value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return <Child label={flag ? "B" : "A"} />;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `child A 1`，再打印 `child B 1`。

## 考察点
- React 会优先按“同一位置 + 同一类型”复用组件实例
- 组件被复用时，内部 state 不会重新初始化
- `useState` 初始值表达式不会在复用时重新生效

## 详细讲解
第一次渲染时，`Child` 接收到的 `label` 是 `A`，所以 `useState` 的初始值是 `1`，输出 `child A 1`。之后 `App` 的 effect 把 `flag` 改成 `true`，父组件重新渲染，但返回的仍然是同一个位置上的 `<Child />`。React diff 时会认为这是同一个组件实例，只是 props 变了，因此不会重新跑 `useState` 初始化逻辑。于是第二次输出就变成了 `child B 1`，说明 `label` 变了，但内部 state 仍然沿用了第一次的值。
