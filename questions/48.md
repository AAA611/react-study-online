# React Output Question 48: Changing key forces a brand new component instance

## Basic Info
- Category: Lists / key
- Difficulty: Intermediate
- Tags: key, remount, state reset
- Type: Read React code and write the output

## Description
`key` 从 `0` 变成 `1` 之后，子组件内部 state 会怎样？

## Code
```jsx
function Child() {
  const [count] = React.useState(100);
  console.log("child", count);
  return null;
}

function App() {
  const [keyValue, setKeyValue] = React.useState(0);

  React.useEffect(() => {
    setKeyValue(1);
  }, []);

  return <Child key={keyValue} />;
}

render(<App />);
```

## Answer
It prints child 100 twice.

## Key Points
- changing key means old instance is discarded and a new one is mounted

## Detailed Explanation
当 `key` 改变时，React 不会复用旧组件实例，而是把它当作一个全新的节点重新挂载。这样一来，子组件内部 state 会重新初始化，所以你会看到 `child 100` 被打印两次。
