# React 输出题 63：没有依赖数组的 useEffect 会在每次渲染后执行

## 基本信息
- 分类：Hooks / useEffect
- 难度：初级
- 标签：useEffect, 无依赖数组, 重渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，控制台会输出什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect", count);

    return () => {
      console.log("cleanup", count);
    };
  });

  return (
    <button onClick={() => setCount(count + 1)}>
      {count}
    </button>
  );
}

render(<App />);
```

## 参考答案
首次挂载时会输出 `render 0 -> effect 0`。点击一次后会继续输出 `render 1 -> cleanup 0 -> effect 1`。

## 考察点
- 没有依赖数组时，effect 会在每次渲染后执行
- 每次重新执行前，旧 effect 的 cleanup 会先运行
- cleanup 读到的是上一次渲染的闭包值

## 详细讲解
第一次渲染时，组件函数先执行，所以输出 `render 0`。提交完成后，effect 执行并输出 `effect 0`。点击按钮后，`count` 变成 `1`，组件重新渲染，于是先输出 `render 1`。由于这个 effect 没有依赖数组，它会在这次渲染后重新执行，而在重新执行之前，React 会先把上一次的 effect 清理掉，所以先输出 `cleanup 0`，再输出新的 `effect 1`。
