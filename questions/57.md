# React 输出题 57：列表头部插入元素但没有稳定 `key` 会导致状态错位

## 基本信息
- 分类：列表 / key
- 难度：中级
- 标签：diff, key, 头部插入
- 题型：阅读 React 代码并写出输出

## 题目描述
列表初始是 `B、C`，随后在头部插入一个 `A`。这里故意使用 `index` 作为 `key`，输出会是什么？

## 代码
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["B", "C"]);

  React.useEffect(() => {
    setList(["A", "B", "C"]);
  }, []);

  return list.map((label, index) => (
    <Item key={index} label={label} />
  ));
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `B B`、`C C`，更新后再打印 `A B`、`B C`、`C C`。

## 考察点
- 使用 `index` 作为 `key` 时，React 会按位置复用节点
- 头部插入元素会让后面的旧实例整体“向后错位”
- 状态错位是列表 diff 中非常典型的问题

## 详细讲解
第一次渲染时，两个 `Item` 分别初始化为 `B` 和 `C`，所以输出 `B B`、`C C`。更新后，列表变成 `A、B、C`，但 `key` 用的是索引。于是 React 会认为位置 `0` 的旧节点还能复用给新的 `A`，所以它内部 state 还是旧的 `B`，输出 `A B`；位置 `1` 的旧节点复用给新的 `B`，内部 state 还是旧的 `C`，输出 `B C`；最后新增一个位置 `2` 的节点，才会真正初始化成 `C C`。这就是典型的状态错位。
