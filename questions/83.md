# React 输出题 83：给可移动节点加上稳定 key 后，位置变化也能保留状态

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：key, 位置变化, 状态保留
- 题型：阅读 React 代码并写出输出

## 题目描述
和上一题几乎一样，但这次给 `Box` 和 `span` 都加了稳定 key。控制台输出是什么？

## 代码
```jsx
function Box({ label }) {
  const [value] = React.useState(label === "A" ? 1 : 999);
  console.log("box", label, value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag ? (
    <>
      <span key="title">title</span>
      <Box key="box" label="B" />
    </>
  ) : (
    <>
      <Box key="box" label="A" />
      <span key="title">title</span>
    </>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `box A 1`，更新后再打印 `box B 1`。

## 考察点
- 稳定 key 可以让 React 在兄弟节点重排时找到同一个实例
- 位置改变不一定会导致重新挂载，关键在于身份是否稳定
- 复用实例时，内部 state 会保留下来

## 详细讲解
第一次渲染时，`Box` 输出 `box A 1`。更新后，虽然 `Box` 的位置从前面挪到了后面，但它的 key 始终是 `"box"`，所以 React diff 时可以直接认出“这还是原来那个 `Box`”。因此旧实例会被复用，内部 state 不会重新初始化，仍然保留第一次的值 `1`。于是第二次输出是 `box B 1`，说明 props 里的 `label` 变了，但 state 没有重置。
