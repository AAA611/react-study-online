# React Output Question 32: setInterval plus stale closure

## Basic Info
- Category: Hooks / closures
- Difficulty: Intermediate
- Tags: setInterval, closure, stale value
- Type: Read React code and write the output

## Description
这个 interval 会一直打印什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    const id = setInterval(() => {
      console.log(count);
      setCount(c => c + 1);
    }, 1000);

    return () => clearInterval(id);
  }, []);

  return null;
}

render(<App />);
```

## Answer
It keeps printing 0.

## Key Points
- the callback reads stale state from its closure
- functional update fixes the state update but not the log value

## Detailed Explanation
定时器回调是在首次渲染时创建的，所以里面读到的 `count` 永远是 `0`。虽然函数式更新能够保证 state 自己不断递增，但 `console.log(count)` 读取的仍然是旧闭包中的变量，因此会一直打印 `0`。
