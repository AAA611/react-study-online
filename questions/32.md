# React 输出题 32：setInterval 与旧闭包

## 基本信息
- 分类：Hooks / 闭包
- 难度：中级
- 标签：setInterval, 闭包, 旧值
- 题型：阅读 React 代码并写出输出

## 题目描述
这个 interval 会一直打印什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    const id = setInterval(() => {
      console.log(count);
      setCount(c => c + 1);
    }, 1000);

    return () => clearInterval(id);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
它会一直重复打印 `0`。

## 考察点
- 回调读取到的是闭包中的旧状态
- 函数式更新能修正状态更新，但不会修正日志里读到的旧值

## 详细讲解
定时器回调是在首次渲染时创建的，所以里面读到的 `count` 永远是 `0`。虽然函数式更新能够保证 state 自己不断递增，但 `console.log(count)` 读取的仍然是旧闭包中的变量，因此会一直打印 `0`。
