# React 输出题 54：父级 DOM 标签类型变化会导致子树重建

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, DOM 类型, 子树重建
- 题型：阅读 React 代码并写出输出

## 题目描述
外层从 `div` 变成 `section` 后，里面的 `Child` 会不会被保留？控制台输出顺序是什么？

## 代码
```jsx
function Child() {
  const [count] = React.useState(100);

  React.useEffect(() => {
    console.log("mount");

    return () => {
      console.log("unmount");
    };
  }, []);

  console.log("child", count);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag ? (
    <section>
      <Child />
    </section>
  ) : (
    <div>
      <Child />
    </div>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：`child 100 -> mount -> child 100 -> unmount -> mount`。

## 考察点
- 宿主节点类型变化时，React 会替换整棵对应子树
- 外层 DOM 类型变化会影响内部组件是否能被复用
- 子树被替换时会触发旧组件卸载和新组件挂载

## 详细讲解
第一次渲染时，`Child` 被放在 `div` 下面，所以先输出 `child 100`，提交后执行 effect，输出 `mount`。当 `flag` 变成 `true` 后，外层节点从 `div` 变成了 `section`。在 React 看来，这不是“同一个节点改了属性”，而是“根节点类型变了”，所以原来的 `div` 子树会整棵被卸载，然后重新创建 `section` 子树。新的 `Child` 会再次渲染，输出第二次 `child 100`；提交时先清理旧实例，输出 `unmount`，再执行新实例的 effect，输出新的 `mount`。
