# React Output Question 15: Unmounting and remounting resets local state

## Basic Info
- Category: Lifecycle / mount and unmount
- Difficulty: Junior
- Tags: conditional rendering, unmount, state reset
- Type: Read React code and write the output

## Description
子组件先被隐藏再重新显示，完整输出顺序是什么？

## Code
```jsx
function Child() {
  const [count] = React.useState(0);

  React.useEffect(() => {
    console.log("mount");
    return () => console.log("unmount");
  }, []);

  console.log("child render", count);
  return null;
}

function App() {
  const [show, setShow] = React.useState(true);

  React.useEffect(() => {
    setShow(false);
    setTimeout(() => setShow(true), 0);
  }, []);

  return show ? <Child /> : null;
}

render(<App />);
```

## Answer
Output: child render 0 -> mount -> unmount -> child render 0 -> mount.

## Key Points
- rendering null unmounts the component
- a remount creates a fresh state bucket

## Detailed Explanation
当 `show` 变成 `false` 时，`Child` 被彻底卸载，所以会先执行 cleanup 输出 `unmount`。随后 `show` 再次变成 `true` 时，React 挂载的是一个全新的 `Child` 实例，而不是恢复旧实例，因此 state 会重新从初始值开始，重新输出 `child render 0` 和 `mount`。
