# React 输出题 15：卸载再挂载会重置局部状态

## 基本信息
- 分类：生命周期 / 挂载与卸载
- 难度：初级
- 标签：条件渲染, 卸载, 状态重置
- 题型：阅读 React 代码并写出输出

## 题目描述
子组件先被隐藏再重新显示，完整输出顺序是什么？

## 代码
```jsx
function Child() {
  const [count] = React.useState(0);

  React.useEffect(() => {
    console.log("mount");
    return () => console.log("unmount");
  }, []);

  console.log("child render", count);
  return null;
}

function App() {
  const [show, setShow] = React.useState(true);

  React.useEffect(() => {
    setShow(false);
    setTimeout(() => setShow(true), 0);
  }, []);

  return show ? <Child /> : null;
}

render(<App />);
```

## 参考答案
输出顺序是：`child render 0 -> mount -> unmount -> child render 0 -> mount`。

## 考察点
- 渲染 `null` 会让组件卸载
- 重新挂载会得到一份全新的本地状态

## 详细讲解
当 `show` 变成 `false` 时，`Child` 被彻底卸载，所以会先执行 cleanup 输出 `unmount`。随后 `show` 再次变成 `true` 时，React 挂载的是一个全新的 `Child` 实例，而不是恢复旧实例，因此 state 会重新从初始值开始，重新输出 `child render 0` 和 `mount`。
