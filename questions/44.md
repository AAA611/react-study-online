# React 输出题 44：事件处理中的日志先于重渲染日志

## 基本信息
- 分类：事件 / 渲染顺序
- 难度：初级
- 标签：事件, console.log, 顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
点击一次按钮，日志顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  return (
    <button
      onClick={() => {
        console.log("handler start", count);
        setCount(count + 1);
        console.log("handler end", count);
      }}
    >
      {count}
    </button>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> handler start 0 -> handler end 0 -> render 1`。

## 考察点
- 事件处理函数会先同步执行完，React 再处理重渲染

## 详细讲解
点击时事件回调会先同步执行完，所以两个 `handler` 日志会先打印出来。`setCount` 只是安排一次新的渲染，并不会中断当前函数执行，因此 `render 1` 会在事件处理结束后才出现。
