# React Output Question 44: Event handler logs happen before rerender logs

## Basic Info
- Category: Events / render order
- Difficulty: Junior
- Tags: events, console.log, order
- Type: Read React code and write the output

## Description
点击一次按钮，日志顺序是什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  return (
    <button
      onClick={() => {
        console.log("handler start", count);
        setCount(count + 1);
        console.log("handler end", count);
      }}
    >
      {count}
    </button>
  );
}

render(<App />);
```

## Answer
Output: render 0, then after clicking handler start 0 -> handler end 0 -> render 1.

## Key Points
- event handlers finish synchronously before React processes the rerender

## Detailed Explanation
点击时事件回调会先同步执行完，所以两个 `handler` 日志会先打印出来。`setCount` 只是安排一次新的渲染，并不会中断当前函数执行，因此 `render 1` 会在事件处理结束后才出现。
