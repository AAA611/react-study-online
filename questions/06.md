# React 输出题 06：setTimeout 中的闭包旧值问题

## 基本信息
- 分类：闭包 / Hooks
- 难度：中级
- 标签：闭包, 定时器, 旧状态
- 题型：阅读 React 代码并写出输出

## 题目描述
按钮点击一次后，1 秒后控制台会打印什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const handleClick = () => {
    setTimeout(() => {
      console.log(count);
    }, 1000);
    setCount(count + 1);
  };

  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## 参考答案
最终打印的是 `0`。

## 考察点
- 定时器回调会捕获旧渲染中的值
- 状态更新不会修改旧闭包中的变量

## 详细讲解
`setTimeout` 回调是在 `count = 0` 的那次渲染中创建的，因此它捕获到的也是旧闭包中的 `count = 0`。点击按钮后虽然页面会更新成 `1`，但已经创建好的定时器回调不会自动拿到新值，所以 1 秒后依然打印 `0`。
