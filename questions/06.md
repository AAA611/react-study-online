# React Output Question 06: Stale closure inside setTimeout

## Basic Info
- Category: Closures / Hooks
- Difficulty: Intermediate
- Tags: closure, timeout, stale state
- Type: Read React code and write the output

## Description
按钮点击一次后，1 秒后控制台会打印什么？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const handleClick = () => {
    setTimeout(() => {
      console.log(count);
    }, 1000);
    setCount(count + 1);
  };

  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## Answer
It logs 0.

## Key Points
- timeout callback captures old render values
- state updates do not mutate old closures

## Detailed Explanation
`setTimeout` 回调是在 `count = 0` 的那次渲染中创建的，因此它捕获到的也是旧闭包中的 `count = 0`。点击按钮后虽然页面会更新成 `1`，但已经创建好的定时器回调不会自动拿到新值，所以 1 秒后依然打印 `0`。
