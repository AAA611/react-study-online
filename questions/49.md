# React Output Question 49: State initialized from props does not automatically follow later props

## Basic Info
- Category: State / props initialization
- Difficulty: Intermediate
- Tags: props, initialization, state
- Type: Read React code and write the output

## Description
父组件把 `value` 从 `1` 改成 `2` 之后，子组件内部 state 会自动同步吗？

## Code
```jsx
function Child({ value }) {
  const [inner] = React.useState(value);
  console.log("child", value, inner);
  return null;
}

function App() {
  const [value, setValue] = React.useState(1);

  React.useEffect(() => {
    setValue(2);
  }, []);

  return <Child value={value} />;
}

render(<App />);
```

## Answer
Output: child 1 1, then child 2 1.

## Key Points
- useState only uses the initial prop value once

## Detailed Explanation
`useState(value)` 只会在首次渲染时把 `props.value` 用作初始值。父组件后续把 `value` 更新为 `2` 时，子组件虽然拿到了新的 props，但内部 state `inner` 不会自动跟着变，因此结果是 `child 2 1`。
