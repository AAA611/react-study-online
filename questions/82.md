# React 输出题 82：没有 key 时，插入一个不同类型的兄弟节点会让组件重新挂载

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：兄弟节点, 无 key, 节点类型
- 题型：阅读 React 代码并写出输出

## 题目描述
第一次 `Box` 在第一个位置，更新后前面插入了一个 `span`。因为没有 `key`，最终输出是什么？

## 代码
```jsx
function Box({ label }) {
  const [value] = React.useState(label === "A" ? 1 : 999);
  console.log("box", label, value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag ? (
    <>
      <span>title</span>
      <Box label="B" />
    </>
  ) : (
    <>
      <Box label="A" />
      <span>title</span>
    </>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `box A 1`，更新后再打印 `box B 999`。

## 考察点
- 没有 `key` 时，React 会优先按位置和类型比较兄弟节点
- 前面插入不同类型节点时，后面的组件可能失去复用机会
- 组件一旦重新挂载，内部 state 会重新初始化

## 详细讲解
第一次渲染时，`Box` 位于第一个位置，所以输出 `box A 1`。更新后，新的第一个位置变成了 `span`，而旧的第一个位置是 `Box`，类型不一样，因此 React 不会把它们当成同一个节点。新的第二个位置又是 `Box`，旧的第二个位置却是 `span`，类型依然不同，所以这个 `Box` 也不会复用旧实例，而是新建一个实例。于是它会重新初始化 state，输出 `box B 999`。
