# React Output Question 36: Child cleanup order when props change

## Basic Info
- Category: Parent child / effects
- Difficulty: Intermediate
- Tags: cleanup, props, effect order
- Type: Read React code and write the output

## Description
当 `value` 从 `0` 变成 `1` 时，完整输出顺序是什么？

## Code
```jsx
function Child({ value }) {
  React.useEffect(() => {
    console.log("effect", value);
    return () => console.log("cleanup", value);
  }, [value]);

  console.log("render child", value);
  return null;
}

function App() {
  const [value, setValue] = React.useState(0);

  React.useEffect(() => {
    setValue(1);
  }, []);

  return <Child value={value} />;
}

render(<App />);
```

## Answer
Output: render child 0 -> effect 0 -> render child 1 -> cleanup 0 -> effect 1.

## Key Points
- render happens before cleanup and new effect commit work
- cleanup still sees the previous value

## Detailed Explanation
子组件先用旧 props 渲染并执行 `effect 0`。父组件更新后，子组件会先根据新 props 渲染出 `render child 1`。提交阶段 React 先清理旧 effect，所以输出 `cleanup 0`，然后再执行依赖新 `value` 的 `effect 1`。
