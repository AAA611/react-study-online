# React 输出题 36：props 变化时子组件 cleanup 的执行顺序

## 基本信息
- 分类：父子组件 / effect
- 难度：中级
- 标签：清理函数, props, effect 顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
当 `value` 从 `0` 变成 `1` 时，完整输出顺序是什么？

## 代码
```jsx
function Child({ value }) {
  React.useEffect(() => {
    console.log("effect", value);
    return () => console.log("cleanup", value);
  }, [value]);

  console.log("render child", value);
  return null;
}

function App() {
  const [value, setValue] = React.useState(0);

  React.useEffect(() => {
    setValue(1);
  }, []);

  return <Child value={value} />;
}

render(<App />);
```

## 参考答案
输出顺序是：`render child 0 -> effect 0 -> render child 1 -> cleanup 0 -> effect 1`。

## 考察点
- 重新渲染会先发生，之后才是 cleanup 和新的 effect
- cleanup 读到的仍然是上一次的值

## 详细讲解
子组件先用旧 props 渲染并执行 `effect 0`。父组件更新后，子组件会先根据新 props 渲染出 `render child 1`。提交阶段 React 先清理旧 effect，所以输出 `cleanup 0`，然后再执行依赖新 `value` 的 `effect 1`。
