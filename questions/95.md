# React 输出题 95：依赖变化时 cleanup 会在新 effect 之前执行

## 基本信息
- 分类：执行顺序 / cleanup
- 难度：中级
- 标签：useEffect, cleanup, 更新顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
`count` 从 `0` 更新为 `1` 时，旧 effect 的 cleanup 和新 effect 的顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect", count);

    if (count === 0) {
      setCount(1);
    }

    return () => {
      console.log("cleanup", count);
    };
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> effect 0 -> render 1 -> cleanup 0 -> effect 1`。

## 考察点
- 依赖变化时，旧 effect 会先 cleanup
- cleanup 读到的是上一次渲染闭包里的值
- 新 effect 会在旧 cleanup 完成后再执行

## 详细讲解
初次挂载时先输出 `render 0`，提交后 effect 输出 `effect 0`，并把 `count` 更新成 `1`。接下来组件重新渲染，输出 `render 1`。因为依赖数组里的 `count` 发生了变化，React 需要处理旧 effect，于是先执行它的 cleanup，输出 `cleanup 0`。注意这里仍然拿到旧闭包里的 `count=0`。等旧 effect 清理完成后，新的 effect 才开始执行，所以最后输出 `effect 1`。
