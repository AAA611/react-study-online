# React 输出题 91：事件处理函数中的同步日志总是先于重渲染日志

## 基本信息
- 分类：执行顺序 / 事件与渲染
- 难度：中级
- 标签：事件处理, setState, 渲染顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
点击一次按钮后，这段代码的控制台输出顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  return (
    <button
      onClick={() => {
        console.log("click start", count);
        setCount(count + 1);
        console.log("click end", count);
      }}
    >
      {count}
    </button>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `render 0`。点击一次后会继续输出 `click start 0 -> click end 0 -> render 1`。

## 考察点
- 事件处理函数会先同步执行完
- `setState` 不会立刻修改当前闭包里的 state 值
- React 会在事件处理完成后再安排新的渲染

## 详细讲解
初次挂载时，组件函数执行一次，所以先输出 `render 0`。点击按钮后，事件处理函数同步开始执行，先输出 `click start 0`。虽然中间调用了 `setCount(count + 1)`，但这只是告诉 React “下一次渲染请把 count 改成 1”，并不会立刻改变当前这次事件回调闭包里的 `count`。因此随后仍然输出 `click end 0`。等事件处理函数执行完成后，React 才会进行新的渲染，所以最后输出 `render 1`。
