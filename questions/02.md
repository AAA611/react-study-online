# React 输出题 02：同一事件中两次 setState 读取的是同一个快照

## 基本信息
- 分类：状态 / 批处理
- 难度：初级
- 标签：批处理, setState, 事件
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮一次后，控制台会输出什么？按钮文本最终会变成多少？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const handleClick = () => {
    setCount(count + 1);
    setCount(count + 1);
    console.log("clicked", count);
  };

  console.log("render", count);
  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## 参考答案
首次渲染会打印 `render 0`。点击一次后，会先打印 `clicked 0`，随后组件重新渲染并打印 `render 1`，按钮上的文本也会变成 `1`。

## 考察点
- 事件处理函数读取的是当前这次渲染的状态快照
- 多个更新会被批量处理
- 这里连续两次 `setCount(count + 1)` 不会直接把结果变成 `2`

## 详细讲解
点击事件里的 `count` 来自当前这次渲染的闭包快照，值仍然是 `0`。因此两次 `setCount(count + 1)` 实际都在设置同一个新值 `1`，而不是先加到 `1` 再加到 `2`。React 会把这两个更新批处理，所以最终只会渲染出 `1`。而 `console.log` 发生在重新渲染之前，所以会先打印 `clicked 0`。
