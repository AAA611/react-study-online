# React Output Question 02: Two setState calls inside one event with same snapshot

## Basic Info
- Category: State / batching
- Difficulty: Junior
- Tags: batching, setState, events
- Type: Read React code and write the output

## Description
点击按钮一次后，控制台会输出什么？按钮文本最终会变成多少？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  const handleClick = () => {
    setCount(count + 1);
    setCount(count + 1);
    console.log("clicked", count);
  };

  console.log("render", count);
  return <button onClick={handleClick}>{count}</button>;
}

render(<App />);
```

## Answer
Initial log: render 0. After one click: clicked 0, then render 1. Button text becomes 1.

## Key Points
- event handlers read the current render snapshot
- multiple updates can be batched
- setCount(count + 1) twice does not become 2 here

## Detailed Explanation
点击事件里的 `count` 来自当前这次渲染的闭包快照，值仍然是 `0`。因此两次 `setCount(count + 1)` 实际都在设置同一个新值 `1`，而不是先加到 `1` 再加到 `2`。React 会把这两个更新批处理，所以最终只会渲染出 `1`。而 `console.log` 发生在重新渲染之前，所以会先打印 `clicked 0`。
