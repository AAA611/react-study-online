# React 输出题 88：在条件分支里显式切换 key 会强制重置同类型组件状态

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：条件渲染, key, 状态重置
- 题型：阅读 React 代码并写出输出

## 题目描述
前后渲染的其实都是同一个 `Panel` 组件类型，但 key 不同。控制台输出是什么？

## 代码
```jsx
function Panel({ label }) {
  const [value] = React.useState(label === "A" ? 1 : 999);
  console.log("panel", label, value);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return flag
    ? <Panel key="B" label="B" />
    : <Panel key="A" label="A" />;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `panel A 1`，更新后再打印 `panel B 999`。

## 考察点
- 对同类型组件来说，key 也会参与身份判断
- 显式切换 key 是一种常见的“强制重置状态”手段
- 同类型不代表一定复用，key 变化同样会打断复用

## 详细讲解
第一次渲染时，`Panel` 使用 key `"A"`，所以初始化 state 为 `1`，输出 `panel A 1`。更新后，虽然组件类型还是 `Panel`，但 key 变成了 `"B"`。这说明 React 需要把它视为一个新的实例，而不是旧实例的更新版本。因此旧的 `Panel` 不会被复用，新的 `Panel` 会重新初始化 state，于是输出 `panel B 999`。
