# React Output Question 11: React.memo with stable primitive props

## Basic Info
- Category: Performance / React.memo
- Difficulty: Intermediate
- Tags: React.memo, props, rerender
- Type: Read React code and write the output

## Description
点击按钮一次后，`MemoChild` 会不会重新渲染？

## Code
```jsx
const MemoChild = React.memo(function Child({ label }) {
  console.log("child render", label);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <div>
      <button onClick={() => setCount(c => c + 1)}>{count}</button>
      <MemoChild label="stable" />
    </div>
  );
}

render(<App />);
```

## Answer
First render: app render 0, child render stable. After one click: app render 1 only.

## Key Points
- React.memo does shallow comparison
- stable primitive props let React skip child rerender

## Detailed Explanation
这里传给子组件的是稳定的字符串 props。`React.memo` 会对 props 做浅比较，发现 `label` 没有变化，因此父组件重新渲染后，子组件会被跳过，不会再次执行。
