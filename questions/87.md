# React 输出题 87：节点从一个父容器移动到另一个父容器时不会被直接复用

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：父容器, key, 节点移动
- 题型：阅读 React 代码并写出输出

## 题目描述
`Item` 看起来只是从左边移动到了右边，而且 key 没变。最终输出是什么？

## 代码
```jsx
function Item() {
  const [count] = React.useState(9);
  console.log("item", count);
  return null;
}

function App() {
  const [move, setMove] = React.useState(false);

  React.useEffect(() => {
    setMove(true);
  }, []);

  return (
    <>
      <section>{move ? null : <Item key="same" />}</section>
      <aside>{move ? <Item key="same" /> : null}</aside>
    </>
  );
}

render(<App />);
```

## 参考答案
会打印两次 `item 9`。

## 考察点
- key 的作用范围只在同一个父节点的同层子元素中
- 节点跨父容器移动时，React 不会把它当成同一个实例直接搬过去
- 这种“移动”通常等价于旧实例卸载、新实例挂载

## 详细讲解
第一次渲染时，`Item` 出现在 `section` 里，所以输出一次 `item 9`。更新后，它从左侧消失，出现在右侧的 `aside` 里。虽然两个地方都写了 `key="same"`，但这两个 key 分别属于不同父节点下的子列表。React 不会跨父节点去匹配和复用子组件，因此这里本质上是左边的旧实例被移除，右边创建了一个新实例，所以会再输出一次 `item 9`。
