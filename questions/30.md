# React 输出题 30：错误边界可以捕获渲染阶段错误

## 基本信息
- 分类：错误处理 / 错误边界
- 难度：中级
- 标签：错误边界, 渲染错误, 降级界面
- 题型：阅读 React 代码并写出输出

## 题目描述
这次错误发生在渲染阶段，控制台会输出什么？

## 代码
```jsx
class Boundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    console.log("boundary");
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      console.log("fallback");
      return null;
    }
    return this.props.children;
  }
}

function Child() {
  throw new Error("boom");
}

function App() {
  return (
    <Boundary>
      <Child />
    </Boundary>
  );
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `boundary`，然后渲染降级界面 `fallback`。

## 考察点
- 错误边界可以捕获后代组件渲染阶段抛出的错误

## 详细讲解
`Child` 在渲染过程中直接抛错，React 会向上找到最近的错误边界。`Boundary` 先通过 `getDerivedStateFromError` 输出 `boundary` 并更新状态，随后重新渲染 fallback 分支，所以又会输出 `fallback`。
