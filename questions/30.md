# React Output Question 30: Error boundaries catch render phase errors

## Basic Info
- Category: Error handling / boundary
- Difficulty: Intermediate
- Tags: error boundary, render error, fallback
- Type: Read React code and write the output

## Description
这次错误发生在渲染阶段，控制台会输出什么？

## Code
```jsx
class Boundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    console.log("boundary");
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      console.log("fallback");
      return null;
    }
    return this.props.children;
  }
}

function Child() {
  throw new Error("boom");
}

function App() {
  return (
    <Boundary>
      <Child />
    </Boundary>
  );
}

render(<App />);
```

## Answer
Output: boundary, then fallback.

## Key Points
- boundaries catch render errors from descendants

## Detailed Explanation
`Child` 在渲染过程中直接抛错，React 会向上找到最近的错误边界。`Boundary` 先通过 `getDerivedStateFromError` 输出 `boundary` 并更新状态，随后重新渲染 fallback 分支，所以又会输出 `fallback`。
