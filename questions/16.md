# React Output Question 16: Missing dependency control can create an infinite loop

## Basic Info
- Category: Hooks / useEffect
- Difficulty: Intermediate
- Tags: infinite loop, effect, dependencies
- Type: Read React code and write the output

## Description
这段代码最终会出现什么现象？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    console.log("effect", count);
    setCount(count + 1);
  });

  console.log("render", count);
  return null;
}

render(<App />);
```

## Answer
It keeps logging render n and effect n until React throws an error like Maximum update depth exceeded.

## Key Points
- effects without dependency arrays run after every render
- an unconditional state update inside such an effect loops forever

## Detailed Explanation
因为 `useEffect` 没有依赖数组，所以每次渲染结束后都会执行一次。effect 里又无条件调用 `setCount`，这会再次触发渲染，于是 `render` 和 `effect` 会不断循环，最后 React 会抛出类似 `Maximum update depth exceeded` 的错误。
