# React 输出题 01：useState 初始值只在首次渲染时生效

## 基本信息
- 分类：Hooks / useState
- 难度：初级
- 标签：useState, 初始状态, 重渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
阅读下面的代码，写出组件首次挂载完成后控制台的输出顺序。

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  console.log("render", count);

  React.useEffect(() => {
    setCount(1);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `render 0`，再打印 `render 1`。

## 考察点
- useState 的初始值只会在首次渲染时使用
- useEffect 会在提交完成后执行
- setState 会触发一次新的渲染

## 详细讲解
第一次渲染时，`count` 的初始值是 `0`，所以先输出 `render 0`。组件完成首次提交后，`useEffect` 执行并调用 `setCount(1)`，React 会安排一次新的渲染，因此随后输出 `render 1`。因为依赖数组为空，所以这个 effect 只会在首次挂载后执行一次。
