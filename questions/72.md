# React 输出题 72：React.memo 配合稳定的基础类型 props 可以跳过重复渲染

## 基本信息
- 分类：性能优化 / React.memo
- 难度：初级
- 标签：React.memo, props, 基础类型
- 题型：阅读 React 代码并写出输出

## 题目描述
父组件更新了自己的状态，但传给子组件的 `title` 一直没变。控制台输出会是什么？

## 代码
```jsx
const Child = React.memo(function Child({ title }) {
  console.log("child render", title);
  return null;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <Child title="React" />
    </>
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render React`。点击一次后只会输出 `app render 1`，不会再次输出 `child render React`。

## 考察点
- React.memo 会对 props 做浅比较
- 稳定的基础类型 props 有助于跳过重复渲染
- 父组件更新不一定会导致记忆化子组件重新执行

## 详细讲解
第一次渲染时，`App` 和 `Child` 都会执行，所以输出 `app render 0` 和 `child render React`。点击按钮后，`App` 因为 `count` 变化重新渲染，输出 `app render 1`。但是 `Child` 被 `React.memo` 包裹，并且这次接收到的 `title` 仍然是相同的字符串 `"React"`。React.memo 做浅比较时会发现 props 没变，于是直接复用上一次的结果，跳过子组件执行，因此不会再次输出 `child render React`。
