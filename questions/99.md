# React 输出题 99：一个 effect 触发更新时，另一个 effect 先拿到的是当前这轮渲染的值

## 基本信息
- 分类：执行顺序 / 多个 effect
- 难度：高级
- 标签：useEffect, 批处理, 当前渲染快照
- 题型：阅读 React 代码并写出输出

## 题目描述
两个 effect 同时存在时，第一个 effect 触发的更新会不会影响第二个 effect 这一轮读到的值？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect A", count);

    if (count === 0) {
      setCount(1);
    }
  }, [count]);

  React.useEffect(() => {
    console.log("effect B", count);
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> effect A 0 -> effect B 0 -> render 1 -> effect A 1 -> effect B 1`。

## 考察点
- 同一轮提交里的多个 effect 都基于当前这轮渲染快照执行
- 前一个 effect 触发的更新，不会立刻改掉后一个 effect 这一轮读到的值
- 更新会体现在下一轮渲染和下一轮 effect 中

## 详细讲解
第一次渲染先输出 `render 0`。提交后，按声明顺序先执行第一个 effect，输出 `effect A 0`，并调用 `setCount(1)`。但这个更新只是为下一轮渲染做准备，不会立刻改变当前提交阶段里其他 effect 看到的值。因此第二个 effect 这一轮仍然读取到 `count=0`，输出 `effect B 0`。随后 React 进入下一轮渲染，输出 `render 1`，然后两个 effect 再分别输出 `effect A 1` 和 `effect B 1`。
