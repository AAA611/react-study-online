# React 输出题 56：删除带稳定 `key` 的中间项不会让其他项状态错位

## 基本信息
- 分类：列表 / key
- 难度：中级
- 标签：diff, key, 删除节点
- 题型：阅读 React 代码并写出输出

## 题目描述
列表初始是 `A、B、C`，随后删除中间的 `B`。每一项都用了稳定的 `key`，控制台输出会是什么？

## 代码
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["A", "B", "C"]);

  React.useEffect(() => {
    setList(["A", "C"]);
  }, []);

  return list.map((label) => <Item key={label} label={label} />);
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `A A`、`B B`、`C C`，更新后再打印 `A A`、`C C`。

## 考察点
- 稳定 `key` 可以让 React 精准定位哪一项被删除了
- 删除中间项时，其他项的组件实例可以继续复用
- `key` 的核心价值是维持列表项的身份稳定

## 详细讲解
第一次渲染时，三项都会各自初始化自己的 state，所以输出 `A A`、`B B`、`C C`。更新后列表变成 `A、C`。因为使用了稳定的 `key`，React diff 时能明确知道 `A` 还是原来的 `A`，`C` 还是原来的 `C`，只有 `B` 被删除了。因此 `A` 和 `C` 不会发生状态串位，第二轮仍然分别输出 `A A` 和 `C C`。
