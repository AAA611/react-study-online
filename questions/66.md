# React 输出题 66：useLayoutEffect 会早于 useEffect 执行

## 基本信息
- 分类：Hooks / useLayoutEffect
- 难度：初级
- 标签：useLayoutEffect, useEffect, 执行时机
- 题型：阅读 React 代码并写出输出

## 题目描述
同一个组件里同时声明 `useLayoutEffect` 和 `useEffect`，首次挂载时控制台输出顺序是什么？

## 代码
```jsx
function App() {
  console.log("render");

  React.useLayoutEffect(() => {
    console.log("layout");
  }, []);

  React.useEffect(() => {
    console.log("effect");
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render -> layout -> effect`。

## 考察点
- 组件渲染总是先于 Hook 副作用执行
- useLayoutEffect 的执行时机早于 useEffect
- 两者都在提交阶段执行，但时机不同

## 详细讲解
组件函数执行时先输出 `render`。提交阶段开始后，`useLayoutEffect` 会更早执行，因此接着输出 `layout`。普通的 `useEffect` 会在更靠后的时机执行，所以最后输出 `effect`。这也是为什么 `useLayoutEffect` 更适合读取布局、同步修正 DOM，而 `useEffect` 更适合做普通副作用。
