# React 输出题 94：useLayoutEffect 会先于 useEffect，并且它们各自创建的定时器也按创建顺序执行

## 基本信息
- 分类：执行顺序 / layout effect
- 难度：高级
- 标签：useLayoutEffect, useEffect, 定时器顺序
- 题型：阅读 React 代码并写出输出

## 题目描述
首次挂载后，这段代码的完整输出顺序是什么？

## 代码
```jsx
function App() {
  console.log("render");

  React.useLayoutEffect(() => {
    console.log("layout");

    setTimeout(() => {
      console.log("layout timeout");
    }, 0);
  }, []);

  React.useEffect(() => {
    console.log("effect");

    setTimeout(() => {
      console.log("effect timeout");
    }, 0);
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render -> layout -> effect -> layout timeout -> effect timeout`。

## 考察点
- `useLayoutEffect` 的执行时机早于 `useEffect`
- 两个 Hook 中创建的异步任务，也会受创建先后顺序影响
- 渲染阶段、提交阶段、异步阶段要分清

## 详细讲解
组件函数先执行，所以最先输出 `render`。提交开始后，`useLayoutEffect` 先执行，因此输出 `layout`，并先注册了一个定时器。然后才轮到 `useEffect`，输出 `effect`，并再注册一个定时器。因为前一个定时器注册得更早，所以通常会先输出 `layout timeout`，再输出 `effect timeout`。这题的重点是把三层顺序拆开看：先 render，再 layout/effect，最后才是异步定时器。
