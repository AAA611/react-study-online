# React 输出题 86：外层容器 key 变化时，里面的子组件 key 也救不回复用

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：父节点, key, 子树替换
- 题型：阅读 React 代码并写出输出

## 题目描述
`Child` 自己有稳定的 `key`，为什么外层 `div` 的 key 一变，它还是会重新挂载？

## 代码
```jsx
function Child() {
  const [count] = React.useState(5);
  console.log("child", count);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  React.useEffect(() => {
    setFlag(true);
  }, []);

  return (
    <div key={flag ? "new" : "old"}>
      <Child key="child" />
    </div>
  );
}

render(<App />);
```

## 参考答案
会打印两次 `child 5`。

## 考察点
- 外层容器一旦失去复用条件，整棵子树都会被替换
- 子组件的 key 只在当前父节点的子列表里生效
- 不能指望内部 key 跨越父级结构变化去保留实例

## 详细讲解
第一次渲染时会输出一次 `child 5`。更新后，外层 `div` 的 key 从 `"old"` 变成 `"new"`，所以 React 认为这是一个全新的父节点。既然父节点都换了，它下面整棵子树也就一起被替换掉了。虽然 `Child` 自己的 key 还是 `"child"`，但这个 key 只在“当前这个父节点的孩子列表”里有意义，没办法跨越父节点的整体替换去保住旧实例，所以会再次输出 `child 5`。
