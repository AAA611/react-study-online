# React 输出题 59：用 `index` 作为 `key` 删除中间项时会发生状态串位

## 基本信息
- 分类：列表 / key
- 难度：中级
- 标签：diff, index key, 删除中间项
- 题型：阅读 React 代码并写出输出

## 题目描述
列表初始是 `A、B、C`，随后删除中间的 `B`。这次 `key` 用的是索引，输出会是什么？

## 代码
```jsx
function Item({ label }) {
  const [value] = React.useState(label);
  console.log(label, value);
  return null;
}

function App() {
  const [list, setList] = React.useState(["A", "B", "C"]);

  React.useEffect(() => {
    setList(["A", "C"]);
  }, []);

  return list.map((label, index) => (
    <Item key={index} label={label} />
  ));
}

render(<App />);
```

## 参考答案
输出顺序是：先打印 `A A`、`B B`、`C C`，更新后再打印 `A A`、`C B`。

## 考察点
- `index key` 会让 React 按位置复用列表项
- 删除中间项时，后面的节点会顶上来并继承错误状态
- 列表一旦可能插入、删除、重排，就不适合用索引当 `key`

## 详细讲解
第一次渲染时，三个 `Item` 初始化后依次输出 `A A`、`B B`、`C C`。更新后列表变成 `A、C`。由于 `key` 还是索引，位置 `0` 的 `A` 继续复用，没有问题；但新的位置 `1` 是 `C`，React 会复用原来位置 `1` 的组件实例，也就是旧的 `B` 那个实例，所以它内部保存的 state 还是 `B`。因此第二轮会输出 `C B`，这正是 `index key` 在删除中间项时最典型的状态串位现象。
