# React 输出题 26：useMemo 可以稳定对象依赖

## 基本信息
- 分类：性能 / useMemo
- 难度：中级
- 标签：useMemo, 稳定对象, effect
- 题型：阅读 React 代码并写出输出

## 题目描述
把对象放进 `useMemo` 后，点击按钮时 effect 还会重新执行吗？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const options = React.useMemo(() => ({ step: 1 }), []);

  React.useEffect(() => {
    console.log("effect");
  }, [options]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## 参考答案
输出顺序是：首次渲染打印 `render 0`、`effect`；点击一次后只会打印 `render 1`，不会再次打印 `effect`。

## 考察点
- useMemo 可以保持对象引用稳定

## 详细讲解
这次 `options` 被 `useMemo` 包裹，依赖数组还是空数组，所以整个组件生命周期里都会复用同一个对象引用。依赖不变，effect 自然也不会因为点击按钮而再次执行。
