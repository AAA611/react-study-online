# React Output Question 26: useMemo can stabilize an object dependency

## Basic Info
- Category: Performance / useMemo
- Difficulty: Intermediate
- Tags: useMemo, stable object, effect
- Type: Read React code and write the output

## Description
把对象放进 `useMemo` 后，点击按钮时 effect 还会重新执行吗？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);
  const options = React.useMemo(() => ({ step: 1 }), []);

  React.useEffect(() => {
    console.log("effect");
  }, [options]);

  console.log("render", count);
  return <button onClick={() => setCount(c => c + 1)}>{count}</button>;
}

render(<App />);
```

## Answer
Output: render 0, effect, then after one click only render 1.

## Key Points
- useMemo can keep object identity stable

## Detailed Explanation
这次 `options` 被 `useMemo` 包裹，依赖数组还是空数组，所以整个组件生命周期里都会复用同一个对象引用。依赖不变，effect 自然也不会因为点击按钮而再次执行。
