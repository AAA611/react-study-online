# React 输出题 98：useMemo 的计算发生在 render 阶段，早于 effect

## 基本信息
- 分类：执行顺序 / render 阶段
- 难度：中级
- 标签：useMemo, render, useEffect
- 题型：阅读 React 代码并写出输出

## 题目描述
首次挂载后，这段代码的输出顺序是什么？

## 代码
```jsx
function App() {
  const value = React.useMemo(() => {
    console.log("compute");
    return "React";
  }, []);

  console.log("render", value);

  React.useEffect(() => {
    console.log("effect");
  }, []);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`compute -> render React -> effect`。

## 考察点
- `useMemo` 的回调发生在 render 阶段
- render 阶段的逻辑会先于 effect 执行
- 不要把 `useMemo` 当成 effect，它本质上仍属于渲染过程

## 详细讲解
组件开始渲染时，`useMemo` 需要先计算出 `value`，所以最先输出 `compute`。拿到结果 `"React"` 之后，组件函数继续往下执行，于是输出 `render React`。等整次渲染提交完成后，`useEffect` 才会执行，因此最后输出 `effect`。这说明 `useMemo` 并不是“挂载后再执行”的副作用，它是在 render 阶段同步运行的。
