# React Output Question 37: useImperativeHandle controls what the parent sees through ref

## Basic Info
- Category: Refs / useImperativeHandle
- Difficulty: Intermediate
- Tags: forwardRef, imperative handle, ref
- Type: Read React code and write the output

## Description
父组件在 effect 里会打印什么？

## Code
```jsx
const Child = React.forwardRef(function Child(props, ref) {
  React.useImperativeHandle(ref, () => ({
    say() {
      return "hello";
    }
  }), []);

  return null;
});

function App() {
  const ref = React.useRef(null);

  React.useEffect(() => {
    console.log(typeof ref.current.say, ref.current.say());
  }, []);

  return <Child ref={ref} />;
}

render(<App />);
```

## Answer
It logs: function hello.

## Key Points
- useImperativeHandle customizes the exposed ref value

## Detailed Explanation
`Child` 并没有把内部 DOM 或实例直接暴露给父组件，而是通过 `useImperativeHandle` 主动返回了一个包含 `say` 方法的对象。所以父组件读到的 `ref.current.say` 是函数，调用结果是 `hello`。
