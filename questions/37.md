# React 输出题 37：useImperativeHandle 决定父组件通过 ref 看到什么

## 基本信息
- 分类：Refs / useImperativeHandle
- 难度：中级
- 标签：forwardRef, 命令式句柄, ref
- 题型：阅读 React 代码并写出输出

## 题目描述
父组件在 effect 里会打印什么？

## 代码
```jsx
const Child = React.forwardRef(function Child(props, ref) {
  React.useImperativeHandle(ref, () => ({
    say() {
      return "hello";
    }
  }), []);

  return null;
});

function App() {
  const ref = React.useRef(null);

  React.useEffect(() => {
    console.log(typeof ref.current.say, ref.current.say());
  }, []);

  return <Child ref={ref} />;
}

render(<App />);
```

## 参考答案
父组件里打印的是：`function hello`。

## 考察点
- useImperativeHandle 可以自定义暴露给父组件的 ref 内容

## 详细讲解
`Child` 并没有把内部 DOM 或实例直接暴露给父组件，而是通过 `useImperativeHandle` 主动返回了一个包含 `say` 方法的对象。所以父组件读到的 `ref.current.say` 是函数，调用结果是 `hello`。
