# React Output Question 29: Error boundaries do not catch event handler errors

## Basic Info
- Category: Error handling / boundary
- Difficulty: Intermediate
- Tags: error boundary, event handler, errors
- Type: Read React code and write the output

## Description
点击按钮时，错误边界会不会渲染 fallback？

## Code
```jsx
class Boundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      console.log("fallback");
      return null;
    }
    return this.props.children;
  }
}

function App() {
  return (
    <Boundary>
      <button onClick={() => { throw new Error("boom"); }}>click</button>
    </Boundary>
  );
}

render(<App />);
```

## Answer
No. The event error escapes the boundary and fallback is not rendered.

## Key Points
- error boundaries do not catch errors from event handlers

## Detailed Explanation
错误边界只能捕获渲染阶段、生命周期和构造函数中的错误。按钮点击时抛出的异常发生在事件处理器里，不属于错误边界的捕获范围，因此 fallback 不会渲染。
