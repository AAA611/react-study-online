# React 输出题 29：错误边界捕获不到事件处理函数中的错误

## 基本信息
- 分类：错误处理 / 错误边界
- 难度：中级
- 标签：错误边界, 事件处理函数, 错误
- 题型：阅读 React 代码并写出输出

## 题目描述
点击按钮时，错误边界会不会渲染 fallback？

## 代码
```jsx
class Boundary extends React.Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }

  render() {
    if (this.state.hasError) {
      console.log("fallback");
      return null;
    }
    return this.props.children;
  }
}

function App() {
  return (
    <Boundary>
      <button onClick={() => { throw new Error("boom"); }}>click</button>
    </Boundary>
  );
}

render(<App />);
```

## 参考答案
不会。这个错误会从事件处理函数里直接抛出，错误边界不会渲染 fallback。

## 考察点
- 错误边界无法捕获事件处理函数里的错误

## 详细讲解
错误边界只能捕获渲染阶段、生命周期和构造函数中的错误。按钮点击时抛出的异常发生在事件处理器里，不属于错误边界的捕获范围，因此 fallback 不会渲染。
