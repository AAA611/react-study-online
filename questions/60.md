# React 输出题 60：给子组件临时加一层包装会让它重新挂载

## 基本信息
- 分类：渲染模型 / diff 算法
- 难度：中级
- 标签：diff, 包装层, 重新挂载
- 题型：阅读 React 代码并写出输出

## 题目描述
第一次直接渲染 `<Child />`，第二次改成 `<div><Child /></div>`。`Child` 会被保留吗？

## 代码
```jsx
function Child() {
  const [count] = React.useState(5);
  console.log("child", count);
  return null;
}

function App() {
  const [wrapped, setWrapped] = React.useState(false);

  React.useEffect(() => {
    setWrapped(true);
  }, []);

  return wrapped ? (
    <div>
      <Child />
    </div>
  ) : (
    <Child />
  );
}

render(<App />);
```

## 参考答案
会打印两次 `child 5`。

## 考察点
- React diff 比较的是整棵返回树的结构
- 节点层级发生变化时，原来的子组件通常无法直接复用
- 给组件临时加一层包装，往往等价于卸载旧实例再挂载新实例

## 详细讲解
第一次渲染时，`App` 直接返回 `<Child />`，所以 `Child` 会初始化并输出一次 `child 5`。更新后，返回结构变成了 `<div><Child /></div>`，树的顶层结构已经不一样了。对 React 来说，这不是“同一个位置上的同一个组件变了 props”，而是“原来这里是 `Child`，现在这里变成了 `div`，`Child` 被放到了它下面”。这种结构变化会让旧的 `Child` 实例失去复用条件，于是旧实例被卸载，新的 `Child` 重新挂载，因此又会再打印一次 `child 5`。
