# React Output Question 23: Cleanup from an empty dependency effect keeps the first value

## Basic Info
- Category: Lifecycle / cleanup
- Difficulty: Intermediate
- Tags: cleanup, closure, unmount
- Type: Read React code and write the output

## Description
子组件卸载时 cleanup 会打印哪个 `value`？

## Code
```jsx
function Child({ value }) {
  React.useEffect(() => {
    return () => {
      console.log("cleanup", value);
    };
  }, []);

  return null;
}

function App() {
  const [show, setShow] = React.useState(true);
  const [value, setValue] = React.useState(0);

  React.useEffect(() => {
    setValue(10);
    setShow(false);
  }, []);

  return show ? <Child value={value} /> : null;
}

render(<App />);
```

## Answer
It prints cleanup 0.

## Key Points
- empty dependency effect keeps the mount-time closure

## Detailed Explanation
这个 cleanup 是在第一次渲染时创建出来的，当时 `value` 还是 `0`。因为 effect 的依赖数组为空，它之后不会重新注册新的 cleanup，所以即使父组件后来把 `value` 改成了 `10`，卸载时 cleanup 里读到的仍然是旧值 `0`。
