# React 输出题 75：内联函数 props 会让 React.memo 子组件重新渲染

## 基本信息
- 分类：性能优化 / useCallback
- 难度：中级
- 标签：函数引用, React.memo, 重复渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
即使 `Child` 用了 `React.memo`，为什么点击后还是会重新渲染？

## 代码
```jsx
const Child = React.memo(function Child({ onClick }) {
  console.log("child render");
  return <button onClick={onClick}>child</button>;
});

function App() {
  const [count, setCount] = React.useState(0);
  console.log("app render", count);

  return (
    <Child onClick={() => setCount((value) => value + 1)} />
  );
}

render(<App />);
```

## 参考答案
首次渲染会输出 `app render 0`、`child render`。点击子组件按钮后会继续输出 `app render 1`、`child render`。

## 考察点
- 每次渲染都会创建新的内联函数
- React.memo 遇到新的函数引用时也会认为 props 变了
- 函数 props 的稳定性同样影响性能优化效果

## 详细讲解
虽然 `Child` 被 `React.memo` 包裹了，但它接收的 `onClick` 是一个写在 JSX 里的内联箭头函数。每次 `App` 渲染时，这个函数都会是一个新的引用。点击按钮后，`setCount` 让 `App` 重渲染，React.memo 比较前后 props 时会发现 `onClick` 不是同一个函数，因此仍然会重新执行 `Child`。所以第二次依旧会输出 `child render`。
