# React 输出题 61：依赖当前 state 的 useEffect 会连续触发到条件结束

## 基本信息
- 分类：Hooks / useEffect
- 难度：中级
- 标签：useEffect, 依赖数组, 状态更新
- 题型：阅读 React 代码并写出输出

## 题目描述
下面这段代码里，`count` 会一路加到几？控制台输出顺序是什么？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  console.log("render", count);

  React.useEffect(() => {
    console.log("effect", count);

    if (count < 2) {
      setCount(count + 1);
    }
  }, [count]);

  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> effect 0 -> render 1 -> effect 1 -> render 2 -> effect 2`。

## 考察点
- 依赖数组中的值变化后，effect 会再次执行
- effect 中更新依赖项本身时，可能形成连续多轮渲染
- 条件判断决定了 effect 是有限次执行还是无限循环

## 详细讲解
首次渲染时，先输出 `render 0`。提交完成后执行 effect，输出 `effect 0`，因为 `count < 2`，所以触发 `setCount(1)`。接着组件重新渲染，输出 `render 1`，然后 effect 再次执行并输出 `effect 1`，又把状态更新为 `2`。第三轮渲染输出 `render 2`，effect 输出 `effect 2`。这时条件 `count < 2` 不再成立，所以不会继续更新，流程停止。
