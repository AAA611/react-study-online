# React Output Question 28: Context with object literal value rerenders consumers

## Basic Info
- Category: Context
- Difficulty: Intermediate
- Tags: context, object reference, provider value
- Type: Read React code and write the output

## Description
为什么 `count` 一直是 `1`，消费者还是会重新渲染？

## Code
```jsx
const Ctx = React.createContext(null);

function Child() {
  const value = React.useContext(Ctx);
  console.log("child", value.count);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  return (
    <Ctx.Provider value={{ count: 1 }}>
      <button onClick={() => setFlag(f => !f)}>{String(flag)}</button>
      <Child />
    </Ctx.Provider>
  );
}

render(<App />);
```

## Answer
Output: child 1 on mount, and child 1 again after a click.

## Key Points
- provider value object gets a new reference every render

## Detailed Explanation
虽然对象里的 `count` 始终是 `1`，但 Provider 每次渲染都会创建一个新的对象字面量。对 React 来说，这就是一个新的 context value，因此消费者会被重新渲染。
