# React 输出题 28：Provider 传入对象字面量会让消费者重渲染

## 基本信息
- 分类：Context / 上下文
- 难度：中级
- 标签：上下文, 对象引用, Provider 值
- 题型：阅读 React 代码并写出输出

## 题目描述
为什么 `count` 一直是 `1`，消费者还是会重新渲染？

## 代码
```jsx
const Ctx = React.createContext(null);

function Child() {
  const value = React.useContext(Ctx);
  console.log("child", value.count);
  return null;
}

function App() {
  const [flag, setFlag] = React.useState(false);

  return (
    <Ctx.Provider value={{ count: 1 }}>
      <button onClick={() => setFlag(f => !f)}>{String(flag)}</button>
      <Child />
    </Ctx.Provider>
  );
}

render(<App />);
```

## 参考答案
首次挂载时会打印 `child 1`；点击一次后仍会再次打印 `child 1`。

## 考察点
- Provider 的对象 value 每次渲染都会得到新引用

## 详细讲解
虽然对象里的 `count` 始终是 `1`，但 Provider 每次渲染都会创建一个新的对象字面量。对 React 来说，这就是一个新的 context value，因此消费者会被重新渲染。
