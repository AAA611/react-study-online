# React Output Question 20: Setting the same state value usually skips a rerender

## Basic Info
- Category: State / bailout
- Difficulty: Junior
- Tags: same value, bailout, render
- Type: Read React code and write the output

## Description
如果按钮把状态重新设置成当前值，会不会再次渲染？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(1);
  console.log("render", count);

  return <button onClick={() => setCount(1)}>click</button>;
}

render(<App />);
```

## Answer
Usually you only see render 1 once. Clicking does not trigger another render.

## Key Points
- React compares next and previous state values
- equal state means no useful update

## Detailed Explanation
当前状态已经是 `1`，点击后再次设置成 `1`，React 比较前后状态发现没有变化，因此通常会直接跳过这次无意义的更新，不会触发新的 `render`。
