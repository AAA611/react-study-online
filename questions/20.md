# React 输出题 20：设置相同 state 通常会跳过重渲染

## 基本信息
- 分类：状态 / 相同值跳过更新
- 难度：初级
- 标签：相同值, 跳过更新, 渲染
- 题型：阅读 React 代码并写出输出

## 题目描述
如果按钮把状态重新设置成当前值，会不会再次渲染？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(1);
  console.log("render", count);

  return <button onClick={() => setCount(1)}>click</button>;
}

render(<App />);
```

## 参考答案
通常只会看到一次 `render 1`。点击按钮不会触发新的有效重渲染。

## 考察点
- React 会比较新旧 state 的值
- 如果新旧值相同，React 会认为这次更新没有实际变化

## 详细讲解
当前状态已经是 `1`，点击后再次设置成 `1`，React 比较前后状态发现没有变化，因此通常会直接跳过这次无意义的更新，不会触发新的 `render`。
