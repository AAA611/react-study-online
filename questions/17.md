# React Output Question 17: Logging state right after setState inside an effect

## Basic Info
- Category: Hooks / useEffect
- Difficulty: Junior
- Tags: effect, stale value, state
- Type: Read React code and write the output

## Description
effect 里先 `setState` 再打印 `count`，输出的是旧值还是新值？

## Code
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    setCount(5);
    console.log("inside effect", count);
  }, []);

  console.log("render", count);
  return null;
}

render(<App />);
```

## Answer
Output: render 0 -> inside effect 0 -> render 5.

## Key Points
- state updates schedule a future render
- the current closure value does not change immediately

## Detailed Explanation
effect 执行时读取到的是当前这次渲染闭包里的 `count`，也就是 `0`。调用 `setCount(5)` 只是安排下一次渲染，不会立刻改掉当前 effect 回调中的 `count`，因此会先打印 `inside effect 0`，随后下一次渲染才输出 `render 5`。
