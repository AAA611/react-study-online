# React 输出题 17：在 effect 里 setState 后立刻读取 state

## 基本信息
- 分类：Hooks / useEffect
- 难度：初级
- 标签：effect, 旧值, 状态
- 题型：阅读 React 代码并写出输出

## 题目描述
effect 里先 `setState` 再打印 `count`，输出的是旧值还是新值？

## 代码
```jsx
function App() {
  const [count, setCount] = React.useState(0);

  React.useEffect(() => {
    setCount(5);
    console.log("inside effect", count);
  }, []);

  console.log("render", count);
  return null;
}

render(<App />);
```

## 参考答案
输出顺序是：`render 0 -> inside effect 0 -> render 5`。

## 考察点
- 状态更新只是安排下一次渲染
- 当前闭包中的值不会立刻变化

## 详细讲解
effect 执行时读取到的是当前这次渲染闭包里的 `count`，也就是 `0`。调用 `setCount(5)` 只是安排下一次渲染，不会立刻改掉当前 effect 回调中的 `count`，因此会先打印 `inside effect 0`，随后下一次渲染才输出 `render 5`。
